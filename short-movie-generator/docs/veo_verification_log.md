### 남은 문제 (다음 보정 목록)
1. **상부 수직 광선 기둥**: 조명=카메라 동축 지시에도 위에서 내리꽂는 빔 생성
   (금지 문구 "no light shafts from above" 무시됨). → 보정안: "the only lights are
   attached beside the camera; nothing above the scene emits or casts light" 강화,
   또는 후처리 상단 비네트로 가림.
2. **해저가 너무 밝고 청록(얕은 바다 톤)**: 심해 해저는 좁은 광원 풀 + 어두운 주변이어야
   함. → 보정안: "only a small pool of light on the seafloor directly ahead; the
   bottom fades to black beyond the beam" + 후처리 그레이딩(② 예정)으로 보정.
3. cut1 초반 하단 검은 띠 잔존(부분) → ②의 cropdetect 자동 크롭으로 처리 예정.

### 결론

  다음 유료 테스트에서 프롬프트 보정 1회로 마무리 판단.

## v5 — 2026-07-05 (핀조명·레이저·밝기 수정, cut1 discovery, ~$0.5)

### 개선 확인 (성공)
- [x] **핀조명(상단 빔) 제거**: 'beam' 어휘 전면 삭제 효과 확인. 위에서 내리꽂는 광선 사라짐.
- [x] **공중 레이저 점 2개 제거**: 사라짐.
- [x] **심해 저노출**: 그레이딩으로 평균밝기 YAVG 23.6→11.3 (약 절반). 실제 심해 ROV 룩 근접.
- [x] ROV 장비 하단 노출·해저 유지.

### ★ 중대 회귀 (미해결)
- **공기 기포 기둥 재발생**: 프롬프트가 기포를 강하게 금지("absolutely NO air bubbles")했음에도
  Veo가 우측에 큰 상승 기포 플룸을 생성. → 물리적으로 불가능한 현상이 다시 등장.
  - 원인: "수중=기포"는 Veo의 매우 강한 prior. 텍스트 부정문만으론 확률적으로 못 막음.
  - 그레이딩이 어둡게 눌러 완화되나 근본 해결 아님.

### 판단 (다음 수)
1. **재생성(re-roll)**: Veo는 확률적 → 다른 시드는 기포 없이 나올 수 있음. cut1 1회(~$0.5)로 확인.
2. **비전 검증+자동 재생성 루프(#5)**: 프레임을 비전 모델로 판정(기포/핀조명/형태) → 위반 시
   자동 재생성. 프롬프트 한계를 구조로 보완하는 근본책. 배치 생산 전 필수.
   → bubbles가 "프롬프트만으론 불충분"을 실증했으므로, #5를 앞당길 근거가 생김.

### 부분 개선 / 미해결
- **기포(문제2): 긍정 프롬프트로도 완전 제거 실패.** bubble/air 단어를 전부 뺐는데도
  초반 프레임에서 해저 바닥으로부터 밝은 상승 플룸 발생. 그레이딩이 상당 부분 어둡게 눌러
  후반/중반 프레임에선 옅은 실트 헤이즈 수준으로 완화되나, 초반엔 밝은 기포가 보임.

### ★ 핵심 발견 (다음 레버)
- **기포가 '해저 바닥'에서 발생.** benthic(해저 노출) 세팅이 트리거로 의심됨
  (모델이 해저 ROV = 스러스터로 인한 퇴적물/기포로 연상).
- **가설 검증안(무료·정확성 유지): pelagic(개방 수층) 프레이밍 테스트.** 덤보문어는 실제로
  해저 '위'를 호버링하므로 해저를 화면에 안 넣어도 정확함 → 해저 트리거 제거 시 기포 소멸 기대.
  트레이드오프: 사용자가 선호한 '해저 배경'을 포기.
- 그 외: 그레이딩 강화(초반 구간 추가 크러시) / 재생성(확률적) / 비전 검증 루프(#5, 유료).
