# CLAUDE.md — 프로젝트 컨텍스트 (Claude Code 자동 로드용)

> Claude Code가 매 세션 자동으로 읽는 프로젝트 규칙 파일
> 상세 요구사항: `PRD.md` / 기술 상세: `docs/spec.md`

## 프로젝트 한 줄
- **카테고리별로 소싱(소재 확보)과 제작 방식을 달리하는, 확장형 9:16 쇼츠/릴스 자동 생성 플랫폼.**
- 하나의 **공용 코어 파이프라인** 위에서, 카테고리마다 소싱처·스타일·제작법만 갈아끼운다.
- 카테고리는 **한 번에 하나씩** 플러그인처럼 추가한다. **첫 번째 카테고리 = 심해 해양생물(deep_sea)**.

## 큰 구조 (Platform Architecture) ★핵심 설계

### 1) 2계층 설계 — 공용 코어 + 카테고리 모듈
| 계층 | 성격 | 포함 (예) |
|---|---|---|
| **공용 코어 (Core)** | 모든 카테고리 불변 | 파이프라인 오케스트레이션 · 라이선스 게이트(룰 엔진) · **시각화 인터페이스 계약** · 합성(concat) · 오버레이(텍스트·워터마크·크레딧) · 오디오 · 출력/QC · 시크릿·크레딧 자동삽입 |
| **카테고리 모듈 (Category)** | 카테고리마다 다름 | 소재 소싱처 · 정보 소스 · 상황/컷 뱅크 · 스타일 프로파일 · 정확성 규칙 · 훅·캡션 톤 |

> 원칙: 파이프라인(코어)은 카테고리 내부를 모른다. 카테고리 모듈이 **정해진 계약(입출력 규격)** 만 채우면 코어가 그대로 굴린다 → 새 카테고리 추가 시 코어 재건축 불필요.

### 2) AI 역할 분담
- **기획·주제선정·전개·시간대별 스크립트·훅**: Claude API (텍스트 지능)
- **캡션·화면 정보 텍스트**: Gemini API *(또는 Claude — `[TBD]` 최종 확정)*
- **영상 클립 생성**: Gemini(Veo) img2video — 카테고리가 제공한 프롬프트로 생성
- **조립**: FFmpeg (Cloudflare Containers)

### 3) 카테고리 확장 원칙
- 새 카테고리 = **카테고리 모듈 1세트**만 작성(코어는 재사용). 아래 "새 카테고리 체크리스트" 충족.
- 카테고리는 **레지스트리**(`registry`)로 등록하고, 실행 시 `category_id`로 해당 모듈을 로드한다.
- 카테고리마다 소싱처·스타일·정확성 규칙이 다르므로, **카테고리별 하드 룰**을 각 모듈에 명시한다.

### 4) 새 카테고리 체크리스트 (New Category Checklist)
새 카테고리를 추가할 때 반드시 정의:
1. **소싱처 + 라이선스 정책** (통과 라이선스 소스 목록, 차단 소스)
2. **정보 소스** (사실 데이터를 어디서 가져와 재작성할지)
3. **상황/컷 뱅크** (제작 대상별 상황 → 컷 프롬프트 매핑 + 정확성 플래그)
4. **스타일 프로파일** (무드·카메라·색·금지 연출)
5. **정확성/윤리 규칙** (카테고리 고유의 "날조 금지" 기준)
6. **훅·캡션 톤** (시청자·포지셔닝에 맞는 카피 규칙)

## 현재 활성 카테고리: `deep_sea` (심해 해양생물) — Phase 1, MVP v0.3

### 한 줄 (전략 확정 v3: 나레이션 야생 다큐 `narrated_wildlife` — What On Earth TV 벤치마킹)
- **실제 심해 생물의 단일 개체 감정 서사** 를 **깊고 거친 ASMR 나레이션 + 단어별 카라오케 자막 +
  동적 야생다큐 영상**으로 전하는 9:16 쇼츠(40초 내외). 팔로워·저장·수익 목표.
- 서사 구조 고정: **[호기심 훅] → [시그니처 실제 행동] → [검증된 팩트 2~3개] → [감정적 마무리].**
- 이전 방향 폐기: **ROV/HUD·스캔 리빌 연출은 폐기**(narrated_wildlife로 전환). 게시물(카드뉴스)은
  당분간 중단(`generate_post=False`). 코드·자산은 보존(롤백/재개 가능).
- 신뢰 안전판(불변): 생물 형태·행동은 실제만. 없는 행동·수치·위험·포식·발광 날조 금지. 감정은 연출·나레이션이 담당.

### 현재 목표 (수직 슬라이스 1개 — `run_narrated`)
- `종명 입력 → 정보 → 소싱(종 특정 실사) → 라이선스 게이트 → 대본(script) → Veo 동적 컷(수심 인지) →
  합성 → TTS 나레이션(문장별 톤) → 단어별 카라오케 자막 → 앰비언트 SFX(배경음악 없음) → 9:16 mp4 + 한국어 캡션`
- 시각화 구현체: **`veo_text2video`** (동적 야생다큐 프롬프트). 키 없으면 `panzoom` 폴백(무나레이션 시 앰비언트만).
- **★기준 이미지 규칙 (비용 최소화 · 재굴림 금지)**: Veo 씨앗용 9:16 기준 이미지는 **Gemini 이미지 1회만**
  생성한다(원본 실사→동물 보존+배경 확장). 구도가 조금 안 맞으면 **유료 재굴림 금지 — 무료 코드로 재배치**한다
  (피사체 무게중심을 화면 중앙 부근으로 shift + 노출 가장자리 경계복제 채움, 순수 PIL). 완벽 정중앙 불필요.
- **★핵심 규칙 (panzoom ↔ Veo 분리 · 절대 위반 금지)**: panzoom(무료 켄번즈)과 Veo(유료)는 **상호 배타**다.
  **panzoom으로 가공/크롭한 이미지를 Veo 씨앗으로 절대 쓰지 않는다**(유료 Veo를 panzoom 이미지로 만들지 않음).
  Veo 씨앗은 **오직 원본 실사 또는 Gemini가 원본에서 만든 기준 이미지**만 사용한다. 컷1 Veo 검증 실패 시엔
  ① Gemini 씨앗/프롬프트를 고쳐 **Veo만 재시도**하거나 ② **panzoom을 최종 무료 렌더로** 쓴다(그 경우 그 편은 Veo 없이 완성).
  즉 `panzoom → 유료 Veo` 경로는 금지.
- 모드: CLI `--mode narrated`(기본) / `hud`(구 ROV 파이프라인, 보존).

### 영상 규격
- **★쇼츠 길이 40초 규칙(운영자 확정 · 강제)**: 총 길이 40초 내외. 시드 종은 27~29초 본문(총 30~39초)로
  정상이나 소싱 종의 LLM 본문이 길게 나와 1분을 넘던 사고 → `pipeline._cap_body_chunks`가 본문
  나레이션을 글자수 예산(165자≈30초, 실측 ja edge-tts 5.4자/초)으로 컷(시드는 안 건드림, 끝맺음 절 보존).
  본문 프롬프트도 '28~32초·14~18절'로 단축.
- **★오프닝 지도·수심 하강 스팅어(운영자 확정 · 플랜 B)**: 오프닝 훅 뒤에 '지도 스캔→해역 락온→실제
  서식수심까지 하강' 애니(9:16, ~2.3초)를 붙인다. 롱폼의 `longform.motion.render_locate_descent`를
  세로·압축 설정으로 재사용(`src/core/reels_stinger.py`). 최종 순서=[훅][스팅어][본문][엔드카드].
  **수심은 종 실제 서식수심(날조 아님), 좌표·정확 숫자는 노출 안 함(하드룰 '임의 좌표 금지' 준수).**
  실패해도 발행 불정지(스팅어 없이 진행). 연결: `pipeline.run_reels` 5.5단계.
  - **★스팅어 효과음 필수(운영자 확정 · 무음 금지)**: 지도 스캔→해역 락온→수심 하강 모션에 맞춰
    **scan/lockon/splash SFX**(롱폼 `longform.sfx`)를 입힌다. 타이밍=scan@0·lockon@`t_map`·
    splash@`t_map+t_flash`. 예전엔 `anullsrc`(무음)이라 지도줌·수심 하강이 소리 없이 지나갔다(지적).
    SFX 생성 실패 시에만 무음 폴백(발행 불정지). 구현: `reels_stinger.build_stinger`.
  - **★서식수심 미상 종은 스팅어 생략(확정 · 날조 방지)**: 종 정보에 실제 서식수심 숫자가 없으면
    (`reels_stinger._depth_max`가 `None`) 지도·수심 하강 애니를 **아예 붙이지 않는다**(가짜 수심으로
    채우지 않음). 이 경우 순서=[훅][본문][엔드카드]. 소싱 종처럼 수심 데이터가 없을 때 흔함.
- **★오프닝 훅 제목 화면밖 넘침 절대 금지(핵심규칙 · 재발방지)**: 오프닝 훅 제목은 **어떤 훅이라도 좌우
  안전영역 안**에 들어와야 한다(오른쪽 경계 밖으로 잘리는 사고 반복 → 금지). 폰트 크기는 반드시
  **'실제로 렌더되는 줄'** 기준으로 자동 축소한다. 특히 어절 3개 이상 2줄 배치일 때 line1은 앞 어절들을
  **가로로 이어붙인(join) 줄**이라, `hook_line1` 하나로만 맞추면 실제 line1이 더 넓어 잘렸다(실제 결함:
  line1="まるで宇宙"로 맞췄으나 렌더는 "まるで宇宙深海の"). → `opening_layout`은 join 폭 기준으로 맞추고,
  과축소되면 어절당 1줄(3줄)로 전환해 큰 글자를 유지한다. 회귀 테스트:
  `test_opening_layout_no_overflow_when_line1_differs_from_join`.
- **★오프닝 홀드(운영자 확정 · 어색함 방지)**: 오프닝 훅에서 **마지막 어절 이펙트가 끝나자마자 본문으로
  넘어가면 어색**하다(특히 3줄 훅). 마지막 어절 완료 시각 뒤에 **최소 0.7초 홀드**를 확보한다
  (`hook_intro_stage._OPEN_HOLD_S=0.7`).
  - **★오프닝 길이 = 나레이션 실제 종료 + 홀드(정확히 · 운영자 확정)**: 오프닝 총 길이는 `apply()`에서
    `opening_seg_s = max(_OPEN_MIN_S, ceil(나레이션끝 + 0.7))`로 **항상** 맞춘다(예전엔 기본 4.6초보다
    클 때만 늘려, 나레이션을 빠르게 해도 총 길이가 4.6로 고정돼 '죽은 홀드'만 늘던 문제 → 수정).
    **홀드(0.7초)는 그대로 두고, 훅 문구가 날아오는 시간(=나레이션)이 짧아지면 오프닝 전체도 그만큼 짧아진다.**
  - **★훅 나레이션 속도(운영자 확정 · 빠르게)**: 훅 TTS는 `hook_tts_rate="+16%"`(빠름), 리드인
    `narr_start_s=0.18`. 문구 팝(어절 등장)은 나레이션 단어경계에 동기되므로, 나레이션이 빨라지면
    문구도 그만큼 빨리 날아온다. (실측: 3줄 훅 나레이션 −14%→+16%로 약 3.9s→2.9s, 홀드 0.7s 유지.)
- **★유튜브 커스텀 썸네일(운영자 확정)**: 유튜브 쇼츠 커스텀 썸네일용으로 **오프닝 훅에서 제목이 전부
  드러난 마지막 홀드 프레임**을 별도 jpg로 저장한다(기본 커버는 t=2s 애니 도중이라 제목이 덜 노출됨).
  `hook_intro_stage.apply(thumb_out=)`가 `render_opening_frames`의 마지막 프레임을 저장 →
  `pipeline.run_reels`가 `output/yt_thumb.jpg`로 배치 → 워크플로가 Release에 `ytthumb_<id>.jpg`로
  올리고 `media.yt_thumb_url` 패치 → 대시보드 쇼츠 상세에 미리보기 + '유튜브 썸네일 저장' 버튼
  (`worker` `saveVideo`를 이미지 대응으로 일반화). 프레임 저장 실패해도 발행 불정지(썸네일만 생략).
- **★고화소 사진 활용(운영자 확정 · 검증 완료)**: 좋은 CC 사진은 영상보다 훨씬 많고 선명하다 → 아래처럼
  **정적 화면·정적 피사체에 사진을 적극 섞는다**. 원칙: **같은 대상 사진만**(오정보 금지), 없으면 자동 폴백
  (발행 불정지), CC-BY는 캡션에 저작자 표기(`写真: …`).
  - **오프닝 훅·엔드카드 배경 = 고화소 원본 사진**(있으면). 정지 화면이라 영상 프레임보다 훨씬 선명·고급.
    `footage.fetch_hero_photo(학명)` → `hook_intro_stage.apply(hero_image=…)`가 `_cover_crop`으로 9:16
    배경 생성. 미확보 시 기존 영상 프레임 폴백.
  - **본문 사진 컷어웨이(반복 피로 완화)**: 소스 영상이 본문보다 짧아 반복될 때만, 같은 대상 고해상 사진
    1~2컷을 본문 중반에 **1.8초·디졸브**로 오버레이(`footage.insert_photo_cutaways`). ★자막 번인 '전'에
    넣어 오디오·자막·길이가 그대로 보존된다. 긴 소스·사진 없음이면 생략(남발·촙핑 방지, 최대 2컷).
  - **켄번즈 줌 다양화**: `footage._kenburns_clip(motion=in/out/pan_l/pan_r, W,H)` — 대상 키 해시로 무브를
    갈라 같은 소재가 반복돼도 매번 똑같지 않게. 16:9(본문 소스)·9:16(컷어웨이) 둘 다 지원.
  - **미세조류 사진 켄번즈**: `_CATALOG["marine_algae"].photo=True` — 영상 부족 시 고해상 사진 후보로 보충
    (현미경 정지사진이 많고 대상도 거의 안 움직여 켄번즈가 잘 맞음). 움직이는 생물 카테고리(deep_sea·
    marine_life)는 사진 보충 안 함(정지컷이 어색 → 영상만). 제작은 `fetch_footage` 켄번즈 재사용.
- 스타일 프로필: **narrated_wildlife** — 자연광(얕은종)/칠흑 어둠(심해종, 수심 인지) + 동적 카메라
  (짧은 와이드 → 결정적 순간 푸시인) + 나레이션 + 단어자막. 화질은 crisp cinematic 지향(거친 질감 미화 폐기).
- **Veo 프롬프트(수심 인지 필수)**: 심해종(≥200m)은 `어둠 + 형태 잠금 + 부정 제약(햇빛·수면·기포·산호초 금지)
  + 단일 탐사광`. 얕은종은 자연광. (스파이크 검증: 원안 그대로면 얕은 산호초로 오도됨 → 수심 인지로 해결.)
- 구조: **3컷** (discovery → behavior → detail), Veo 8초/컷 → FFmpeg concat. 나레이션이 영상보다 길면 마지막 프레임 복제로 연장.
- 카메라(벤치마크 정밀분석 반영): **거의 정지·느린 관찰**. 빠른 팬·빠른 줌·완전 정면 대칭 앵글 금지.
  3/4·측면 앵글, 눈높이, 얕은 심도, 피사체 주위 넉넉한 어둠. **스케일 대비는 한 컷 안 급줌이 아니라
  컷 전환(중간거리 와이드 ↔ 극접사)으로** 만든다.
- **오디오(필수)**: 최종 영상에는 **심해 앰비언트 사운드가 반드시 포함**된다(무음 출력 금지). 단, Veo 네이티브 오디오(`generate_audio=True`, 비용 약 2배·컷마다 소리 상이)로 만들지 않고, **후처리 audio 모듈에서 로열티프리 심해음을 FFmpeg로 레이어링**한다(저비용·전 컷 일관). → raw 클립 생성 단계는 `generate_audio=False`(무음), 앰비언트는 합성 후 입힘.

  담백하게 일상어(노동·제도·생활)로 치환**해서 만든다(들뜨지 않게, 독은 옅게, 마지막은 다시 경외로 착지).

- **★쇼츠 제목 해시태그 2개(운영자 확정 · 강제)**: 모든 쇼츠 제목(`yt_title`·`yt_title_ko`) **끝에
  해시태그 정확히 2개**를 붙인다. 두 태그는 그 영상의 발행 해시태그와 **동일**([내용 태그 1개 + 고정
  공통 1개] = `_final_tags` 결과)을 재사용해 제목·캡션 태그가 일치하게 한다. LLM은 제목 본문에 태그를
  넣지 않고(프롬프트 유지), 시스템이 `_clean_title`로 본문을 정리한 뒤 태그 2개를 결정론적으로 덧붙인다.
  (구현: `rich_caption._title_with_tags` — LLM/폴백 두 반환부에 적용; 대시보드 옛 레코드 폴백은
  `worker/index.mjs` `titleTags`가 레코드의 `hashtags`/`hashtags_ko` 앞 2개를 붙인다.)
- **★주제 피사체 검증(핵심규칙 · 재발방지)**: 영상은 **주제(그 종·그 배)가 실제로 화면에 잡혀야** 한다.
  이름으로 소싱만 하고 피사체가 안 잡힌 영상을 그냥 쓰면 안 된다(실사고: 텐트새우 본문이 '빈 물'만,
  난파선 본문이 '잠수사 준비'만).
  - **Step1(구현 · 빈 화면 배제)**: `footage.subject_visibility`(대구조 밝기분산 p75, 실측 생물≥29·빈물≤11)로
    '아무것도 안 보이는 빈 물/준비 컷'을 게이트(`_MIN_VISIBILITY=16`). 미달이면 소스 폐기 → auto 후보 순회가
    다음(피사체 있는) 클립으로 넘어간다. **난파선은 잠수사 위주 영상이 흔해 그 배의 사진을 찾아 켄번즈로
    우선 대체**(배가 확실히 보임 · `_wreck_photo_footage`). 사진 없으면 영상+가시성 게이트.
  - **Step2(비전 LLM 의미검증) = ★난파선 전용(운영자 확정 · 존폐 사고 재발방지 · 절대 위반 금지)**:
    Step2(`footage.footage_shows_subject`)는 **난파선(다이버 vs 배 구분)에만** 하드 리젝트로 쓴다.
    ★생물에는 절대 종ID로 Step2를 걸지 않는다 — 실사고: 생물에 학명(예: `Holothuria leucospilota`)을
    주면 비전 LLM이 384px 프레임에서 '그 종'인지 시각으로 식별하지 못해 **진짜 생물 영상까지 전부 폐기**,
    소싱한 심해·해양생물이 **100% 제작 실패**했다(sepia·octopus·holothuria·plankton 전부). 종 단위
    시각 식별은 애초에 불가능한 요구다. → **생물은 Step1(가시성)만으로 게이트**한다(빈 물/준비 컷은 Step1이
    배제). 이 분기를 다시 생물에 적용하지 말 것. (구현: `fetch_footage`가 `key.startswith("wreck ")`일 때만
    Step2 하드 리젝트.) **소싱↔제작 게이트 정합**: `discovery.validate_source_url`도 Step1(가시성)을 적용해
    '소싱된 영상 = 제작 가능한 영상'이 되게 한다(Step2 종ID 폐지로 소싱·제작 게이트가 일치).
- **⭐ 심해 적합성 검증 (deep_sea 전용 · 허위사실 방지 · 절대 위반 금지)**: 표층·연안 종이 「심해 도감」에
  잘못 편입돼 **가짜 수심('水深 200〜2,000 m')**이 자막으로 붙던 사고(실사고: #026 정어리 Sardinops
  sagax = 표층 회유어). 재발 방지 규칙:
  - **소싱 게이트**: deep_sea 소싱은 `deepsea_verify.verdict(sci, en, corpus)`를 통과한 종만 채택한다
    (`discover(..., verify=...)`). ① 표층·연안 분류군(정어리·청어·멸치·고등어·참치·연안 두족류 Sepia/
    Octopus vulgaris·군소 Aplysia 등)은 **하드 배제** ② 심해 근거(**서식 최대수심 ≥200 m 문헌 수치** 또는
    **심해대 키워드**: deep-sea·abyssal·bathyal·mesopelagic·hydrothermal vent·深海 …)가 있어야 채택
    ③ 근거가 없으면 배제. 근거는 종(→상위 분류군) 위키 **본문 발췌**(`_habitat_corpus`, 도입부 아님 —
    수심은 서식/분포 문단에 흔함)에서 확인.
  - **수심 날조 금지**: 문헌 수심이 없으면 **지어내지 않는다**. `_parse_depth('')`는 옛 기본값
    `(200,2000)`을 폐기하고 `(0,0)`(=미상)을 돌려주며, 엔드카드·오버레이는 `depth_max<=0`이면 **수심 줄을
    아예 생략**한다(`hook_intro`·`module.build_reels_caption`). 검증이 문헌에서 실제 추출한 수심만 표기.
  - **제작 직전 게이트(2중 방어 · 핵심)**: 소싱을 우회해 data.SPECIES에 들어온 종(수동 입력·구버전
    데이터)이라도, **영상이 렌더되기 직전** `category.verify_subject(info)`가 다시 검증한다
    (`pipeline._verify_subject_or_raise` → narrated·reels·carousel 세 경로 모두). 표층·연안 종(정어리·
    갑오징어 Sepia·참문어 Octopus vulgaris 등)은 **학명만으로 즉시 차단**(PipelineError → 제작 중단),
    그 외는 위키 서식 문헌으로 심해 근거 재확인. → '소싱 통과 여부'와 무관하게 **가짜 영상이 만들어지는
    순간을 막는다**. (실사고: Octopus vulgaris 가 '水深 200〜2,000 m'로 이미 발행됨)
  - **발행 원장 감사**: 이미 만들어진 `catalog.json`도 같은 검증기로 감사해 부적합 발행분을 제거한다
    (정어리·갑오징어 삭제 완료). 원격 라이브러리(Cloudflare)의 발행 영상은 관리자 삭제 버튼으로 제거.
  - 회귀 테스트: `tests/test_deepsea_verify.py`(정어리 배제·연안종 배제·수심추출·키워드 채택·근거없음
    배제 + **제작게이트가 표층종 차단·심해종 통과**).
- **⭐ 피사체 추적 = 색 무관 일반 검출(핵심 결함 수정 · 절대 회귀 금지)**: 예전 `reframe._red_pixels`는
  **적색(r-g)만** 신호로 삼아 붉은 생물만 추적됐다. 그 결과 **바다나리(ウミユリ)·흰 해파리·창백한
  두족류** 등 색이 옅은 주인공은 신호 0이 되어 크롭이 놓치고(화면에서 사라짐) 컷 선택도 빈 배경(암초)

  ③국소 텍스처)으로 색과 무관하게 '구조를 가진 눈에 띄는 덩어리'를 잡는다. 실검증: 바다나리 클립에서
  생물 프레임 점수 ≫ 빈 암초 프레임(옛 적색전용은 반대로 붉은 암초를 최고점으로 오선택).
  - **단일/다수 프레이밍(사용자 규칙)**: `_subject_focus`가 '가장 강한 단일 덩어리 중심 + 분산도(spread)'를
    반환. **1마리(spread 낮음)=중앙 줌인**, **여러 마리(spread≥0.5)=접사 줌을 1.35로 완화해 전체구도를
    남긴다**(개체 사이가 아닌 무리 전체가 화면에 담기게). fit 컷=전체구도, closeup 컷=한 마리 중심.
- **⭐ 캡션 최소 분량 보장(운영자 확정 · 짧아짐 방지)**: LLM 캡션은 ≥260자 강제(재시도)지만 **폴백은
  한두 줄로 빈약**해지던 문제 → `rich_caption._CAPTION_FLOOR=230` 미만이면 상시(evergreen) 서술을
  덧붙여 분량을 채운다(`_pad_to_floor`). 상시 서술은 **종별 거짓 주장 없이** '깊은 바다·관측 기록'이라는
  일반 사실만(날조 금지) · seed 해시로 영상마다 다른 조합. 폴백도 사실을 최대 활용(fun_facts 5개·수심/
  분포/서식지 각각 독립 서술). 해시태그는 정확히 2개[내용1+고정1] 정책 유지.
- **⭐ 심해 부적합 → 일반 해양생물 재분류(폐기 대신 살림)**: deep_sea 소싱에서 `deepsea_verify`에 탈락한
  **표층·연안 종**은 버리지 않고 `marine_life` 후보로 **재분류**하고, 수심은 문헌 실측값으로 바로잡는다

- **소싱(종 특정 필수)**: 이미지는 **'그 종'의 실사**여야 한다. `scientific_name`·영문명으로 **위키미디어
  커먼스 → GBIF**를 조회해 통과 라이선스(PD/CC0/CC-BY) 이미지를 받는다(`_fetch_species_photo`).
  실사 사진 우선(오래된 도판·삽화·박제·지도는 후순위 점수화). 미확보 시에만 합성 플레이스홀더(cc0).

- **정보 소스**: FishBase·WoRMS·GBIF·OBIS·한국 공공 DB → 재작성 후 출처 표기.
- **저작권 표기(캡션 필수)**: 릴스 캡션 말미에 **이미지 출처(저작자·출처·라이선스) + 종 정보 출처**를
  반드시 삽입한다(`copywriter.append_attribution`). CC-BY는 저작자·라이선스 표기가 법적 의무.
- **사실 정확성(생물 왜곡 금지)**: 생물 형태·행동은 실제만. 안 하는 포식·가짜 크기/위험·사람(다이버) 조우·종에 없는 발광·픽션 서사 금지. 감정·긴장은 나레이션·연출이 담당(사실 왜곡 아님).
- **나레이션(narrated_wildlife)**: 대본은 30~45초·5~6문장 현재형 단문, 문장별 톤태그([gravelly][slow]
  [whispered][reverent][awe]). **없는 행동·수치 날조 금지**(종 실제 데이터만, `script.py`). 배경음악 없음,
  앰비언트 SFX만. TTS는 Gemini(깊은 ASMR, Charon). → 무음 발행 금지(나레이션 또는 앰비언트 필수).
- **단어별 자막**: 나레이션에 동기된 단어 팝업(굵게·중앙). Gemini TTS가 단어 타임스탬프를 안 주므로
  문장 구간을 어절 글자수 비례로 근사 분배(`subtitle.py`). 정밀 필요 시 ElevenLabs로 교체 가능(유료).
- **정확성 플래그 준수**: 컷 프롬프트는 상황 뱅크의 `accuracy_flags`(예: `bioluminescent`)에 위배 금지.
- **오디오 필수**: 최종 산출물은 소리(나레이션+앰비언트)를 반드시 포함(무음 발행 금지).
- **[legacy] hud 모드 전용 규칙**: 아래는 `--mode hud`(구 ROV) 경로에만 적용(현 기본은 narrated).
  근접 경보(붉은 '▲ 근접 경보'+쿵쿵)는 실제 근접·인지만·'공격' 날조 금지; ANALYZING 서술 앰버(#FFC24D).
- **도감 번호**: 엔트리 No.는 커밋되는 원장(`catalog.json`)으로 누적(`peek_next`/`log_entry`) —
  CI 컨테이너가 매번 새로 떠도 번호가 리셋되지 않게. 제작 페이지 현황판은 `#000_국문명`으로 표기.
- **★종 중복 제작 절대 금지(재탕 폐지 · 사용자 확정 · 절대 위반 금지)**: `auto` 제작은
  `footage_candidates()`가 **아직 제작 안 한 종만**(제작 원장 `catalog.json`에 없는 종만) 후보로 준다.
  파이프라인은 이 후보를 순서대로 실사 확보 시도해 첫 성공 종으로 제작하되(정지 소스 등 게이트 실패는
  다음 후보로 폴백), **이미 만든 종은 후보에서 영구 제외**한다.
  - 예전엔 풀 소진 시 회차로 **순환(재탕)**해 이미 만든 종을 다시 뽑았다 → 심해붉은해파리 등 **중복 제작
    사고**. 그 순환 로직은 **폐지**했다. 다시 넣지 말 것.
  - **미제작 종이 하나도 없으면**(풀 소진) → ★**자동 발굴로 풀을 스스로 보충**한다(하드룰 #14).
    예전엔 여기서 제작을 중단(정지)하고 운영자가 손으로 시드를 추가해야 했지만, 이제
    `discovery.replenish`가 새 해양생물 영상을 날조 없이 발굴해 넣고 계속 제작한다. (수동 시드 추가도
    여전히 가능하지만 필수는 아니다.)
  - 원장(`catalog.json`)은 반드시 커밋돼야 이 판단이 회차 간 유지된다(워크플로 커밋 스텝 필수). 이 판단은 **커밋된 제작 원장(카테고리별 `*_catalog.json`)**을
  기준으로 하므로, 원장이 커밋되지 않으면 CI가 매번 새 컨테이너로 떠서 "전부 미제작"으로 오판 →
  **항상 첫 대상만 반복 생성**된다(침몰선이 3회 연속 같은 배로 나온 실제 사고). 따라서:
  ① `run_reels`가 성공 시 `log_catalog` 기록(구현됨) ② **워크플로 커밋 스텝이 `src/categories/*/*_catalog.json`을
  반드시 포함**(`generate-short.yml`). ③ 한 카테고리엔 실사영상 확보 대상을 **여러 개** 시드해야 로테이션이 의미가 있다
  (침몰선=아리에스·U-1277·마데이렌스 3척). 새 카테고리 추가 시 이 3가지를 반드시 지킨다.

## 공용 하드 룰 (Core Hard Rules — 전 카테고리 위반 금지)
1. **저작권 게이트**: 통과 라이선스(`public-domain`/`cc0`/`cc-by`/**`cc-by-sa`**/`kogl-type1`)만 사용.
   **`cc-by-nc`(비상업)·`cc-by-nc-sa`·`unknown`은 차단.** 차단 에셋은 시각화 이후 절대 미포함.
   - ★**CC-BY-SA 오픈(운영자 확정)**: 소스 풀 확대를 위해 CC-BY-SA도 허용한다(상업이용 가능·표시 조건 →
     저작자·라이선스 크레딧 필수). 구현 시 주의: `'cc by'`가 `'cc by-nc'`의 부분일치라 **NC를 먼저 걸러야**
     오통과가 안 난다(`footage._norm_license`·`discovery._norm_license`에 NC 선차단 있음 — 제거 금지).
2. **시각화 인터페이스 계약**: 구현체(veo/panzoom 등)는 교체 가능해야 하며 상·하류 모듈은 계약만 의존.
3. **출처 크레딧 자동 삽입**: 모든 출력물에 소스 크레딧(예: NOAA) 자동 포함.
4. **시크릿**: API 키는 `.env`에서만 로드, 하드코딩·커밋 금지.
5. **완성 판정**: 각 슬라이스는 수용 기준 통과 후 다음 진행.
6. **정확성 원칙**: 각 카테고리의 정확성/윤리 규칙을 코어가 강제(게이트로 차단).
7. **화면 텍스트 조판 규칙(모든 온스크린 문구 공통)**:
   - **단어 중간에서 줄바꿈 금지.** 단어가 줄 끝에서 잘리면 그 단어를 통째로 다음 줄로 내린다.
     (구현: HTML 텍스트는 `word-break:keep-all` — 한국어 어절이 음절 단위로 쪼개지지 않게.)
   - **위도우(orphan) 정리.** 마침표 뒤에 1~2단어만 남으면 그 조각을 다음 줄로 넘겨 문단을 매끄럽게.
     (구현: `text-wrap:pretty`.)
   - 단, **텍스트가 프레임(안전영역) 밖으로 넘치지 않는 범위**에서만 재정리한다(넘칠 바엔 줄수 우선).
   - **임의 좌표·수치 금지.** 근거 없는 좌표는 숫자로 쓰지 말고 `LOCATING…` 등 상태 문구로 대체.
     수치는 근거를 둔다(예: 수온=수심 기반 역산 `_deep_sea_temp_c`, 수심=종 실제 서식수심).
   - **★학명 표기 규칙(모든 화면·캡션 공통 · 절대 위반 금지).** 학명(binomial)은 **반드시 이탤릭**으로
     쓰고, **속명(첫 단어) 첫 글자는 무조건 대문자, 종소명은 소문자**로 쓴다(예: *Enypniastes eximia*).
     엔드카드·오프닝·캡션·게시물 등 학명이 노출되는 모든 곳에 적용. (구현: 화면 렌더는 이탤릭 폰트로,
     텍스트 캡션은 마크다운 `*...*`로. 첫 글자 대문자화는 렌더 시 강제.)
8. **★존댓말 규칙(나레이션·자막·캡션 공통 · 절대 위반 금지).** 반말(常体/평서체)을 절대 쓰지 않는다.
   **일본어는 です・ます체(敬体)**, **한국어는 ~습니다·~입니다체(존댓말)**로만 쓴다.
   - 적용 범위: **본문 나레이션 대본·화면 자막·발행 캡션·DM 회신** 등 "문장형 콘텐츠". (엔드카드의 종명·특징 라벨은 체언구라 해당 없음.)
   - 금지 예: 일본어 「〜だ」「〜する」「〜がいる」「〜終える」/ 한국어 「〜다」「〜한다」「〜있다」.
   - 허용 예: 일본어 「〜です」「〜します」「〜がいます」「〜のです」「〜でしょうか」/ 한국어 「〜입니다」「〜합니다」「〜있습니다」.
   - **★예외(핵심 규칙 · 절대 혼동 금지): 오프닝 훅 문구와 유튜브 쇼츠 제목은 존댓말로 만들지 않는다.**
     이 둘은 임팩트가 생명인 마케팅 카피라 존댓말 강제 대상이 아니다 — 짧은 단문·체언종결·의문형·도발형·언리(言い切り) 종결 모두 허용(정중함보다 후킹·속도감 우선).
   - 구현: 대본 생성(`script.py`)·캡션 생성(`rich_caption.py` 캡션부)에 敬体/존댓말 강제. **훅(`deep_sea/hook.py` `_PROMPT`)·제목(`rich_caption.py` yt_title)에는 존댓말 강제를 넣지 않는다.**
9. **★워터마크/로고 제거 규칙(실사 영상 · 절대 위반 금지 · 1초 검증 의무).** 소스 실사(NOAA 등)에 박힌
   로고·글씨·URL·크레딧 슬레이트는 **최종 화면에 절대 노출하지 않는다.**
   - **좌표 하드코딩 금지**: 고정 `logo_box` 하나만 지우는 방식은 폐기됐다(박스보다 넓은 로고,
     하단 중앙 "OCEANEXPLORER.NOAA.GOV" URL, 전면 크레딧 슬레이트가 그대로 노출된 사고 실존).
   - **자동 프로세스(`src/core/watermark_qc.py`) 의무 사용**: ① 원본 클립을 **1초 간격 OCR 스캔**
     ② 크레딧 슬레이트/대형 문구가 있는 초는 **사용 구간에서 회피**(시작점 이동) ③ 남은 워터마크는
     탐지 박스로 **자동 delogo** ④ 렌더 결과를 **다시 1초 간격 검증** → 금지 문구(NOAA·OCEANEXPLORER 등)
     검출 시 박스 보강 1회 재렌더, 그래도 남으면 **제작 실패 처리(발행 차단)**.
   - 적용 지점: 롱폼 세그먼트 본문(`segment.py render_segment`) + 콜드오픈(`compile.py`, 세그먼트가
     확정한 `wm_boxes`·`footage_start` 재사용) + 롱폼 완성본 최종 게이트(`run_longform.py`) +
     릴스 소스 세척·완성본 게이트(`pipeline.py`, 엔드카드의 정당한 크레딧 구간만 검사 제외).
   - **★OCR 비의존 인위 삽입물 검출(운영자 아이디어 · 확정 · 재발방지)**: 스타일화된 로고·타이틀카드
     (예: 'SUBMANIA' 다이빙스쿨 로고)는 **OCR이 그래픽을 못 읽어** 슬레이트 회피가 실패했다(실사고: Batelo).
     → 글자를 '읽지' 말고 **인위적 합성물인지**를 판별한다: 합성 그래픽은 밑 영상이 움직여도 **픽셀-정확
     고정(frozen)** + **쨍한 색/날카로운 엣지(graphic)**를 갖는다. `watermark_qc.detect_insert_seconds`가
     **4초 창**(핵심 — 생물은 4초면 부유물·미세이동으로 frozen이 풀리지만 합성 카드는 4초 내내 고정)으로
     `frozen∧graphic` 중앙밴드 비율을 재, **중앙 5%↑면 그 구간을 회피 초로**(→시작점 이동) 만든다.
     `plan()`이 OCR 슬레이트 초와 이 삽입물 초를 합쳐 시작점을 카드 구간 뒤로 옮긴다. **큰 중앙=SKIP,
     작은 가장자리=기존 delogo**. 실클립 보정: 배텔로 카드 → start 0→10.5s, 깨끗 NOAA(enypniastes) →
     오탐 0. 파라미터(`_INSERT_FROZEN_MAXDIFF=2`·`_INSERT_WIN_S=4.0`·`_INSERT_SKIP_CENTRAL=0.05`)를
     낮추면(2초창 등) 정지한 생물을 카드로 오탐하니 되돌리지 말 것. 회귀 테스트: `test_insert_detect.py`.
   - CI 필수 의존성: `tesseract-ocr`(apt) + `pytesseract`(pip) — 워크플로 설치 스텝에서 제거 금지.
   - **속도('빨리감기' 검수)**: OCR은 프레임당 tesseract 서브프로세스라 느리다 → ①프레임을 **720폭으로
     축소**(워터마크는 큰 글자라 충분히 읽힘) ②**대비강화 단패스**(2패스 폐기) ③프레임들을 **스레드풀
     병렬**(서브프로세스라 GIL 해제 → 실병렬). 실측 1280·2패스 대비 6배↑. 정확도(검출력)는 유지.
     이 3가지를 낮추면(해상도↑·다패스·직렬) 다시 느려지므로 되돌리지 말 것.

10. **★쇼츠(9:16) 자막 크기·분절 규칙.** 세로 영상 가독성을 위해 본문 자막은 크게(`pipeline.REELS_SUB_SCALE`,
    현재 1.8배) 쓴다. 자막이 커져 프레임을 넘칠 땐 **줄바꿈(WrapStyle) 금지 — 대신 카라오케 단위를
    `narration_sync._fit_pieces`로 '한 줄에 들어가는 조각'으로 쪼개 순차 표시**한다(발화 시각은 조각
    글자수 비례로 분배해 싱크 유지). ASS `WrapStyle: 2`(자동 줄바꿈 off) 유지가 이 규칙의 전제.

11. **★정지-이미지 소스 배제 규칙(절대 위반 금지).** 움직임이 없는 '정지 화면(사진을 영상으로 포장한 것,
    거의 멈춘 클립)'은 **소스로 쓰지 않고 영상으로 절대 제작하지 않는다.**
    - 검출: `watermark_qc.motion_score`(원본 1fps 인접 프레임 median 밝기차). 실측 실제 수중 영상 9~36 vs
      정지 이미지 0~1 → 문턱 `_STATIC_THRESHOLD=3.0` 미만이면 정지로 판정(`is_static_source`).
    - 차단 지점: `footage.fetch_footage`(모든 소스의 단일 관문) — 정지 판정 시 `None` 반환 →
      릴스는 제작 중단, 롱폼은 그 종 스킵. 시드 확충 루틴도 후보를 이 기준으로 걸러 추가한다.
    - 이유: 정지 화면은 '영상'의 기본 가치를 무너뜨린다(움직이는 실사만이 채널 정체성).

12. **★소스 품질 게이트(번인 로고·인트로 카드·레터박스 방지 · 절대 위반 금지).** 아마추어 소스
    (다이빙 영상 등)는 인트로 타이틀카드·로고(예: 'SUBMANIA Escola de Mergulho')·아웃트로 크레딧이

13. **★해양생물 쇼츠 컷 리듬 규칙(지루함 방지 · wide=False 전용).** 심해·일반 해양생물 쇼츠는
    한 구간을 길게 보여주면 지루하다 → `reframe_to_vertical(..., wide=False)`가 **씬 분절 + 라운드로빈
    인터리브**로 짧게 자주 전환한다.
    - **씬 분절**: `_detect_scene_cuts`(ffmpeg `select='gt(scene,thr)'`)로 원본의 씬 전환 시각을 찾아
      `_build_regions`로 N개 구간(기본 3~6, 씬 부족 시 4등분 폴백)으로 나눈다.
    - **인터리브 순서(사용자 확정)**: 매 컷마다 다음 씬으로 넘기고 한 바퀴 뒤 그 씬의 '다음 순간'을 쓴다
      → `1-1,2-1,…,N-1, 1-2,2-2,…`(예: N=4면 1-1,2-1,3-1,4-1,1-2,…). 각 컷 ≈2.2초.
      같은 씬의 매 패스는 `_pick_windows_in_range`로 피사체 잘 보이는 '다른 순간'을 고른다(번인 텍스트 배제).
    - **줌인 얼굴 중앙(사용자 확정)**: 접사(줌인) 컷은 크롭 중심을 가장 강한 적색(=ROV 조명 받는 얼굴)의
      무게중심에 두되, 피사체가 프레임 가장자리라 넓은 크롭이 클램프되면 얼굴이 옆으로 밀린다 →
      **얼굴을 정중앙에 둘 만큼만 줌을 더 넣어(상한 1.9배) 클램프를 피한다**(과확대는 금지).
    - **난파선(wide=True)은 예외**: 선체 전체 원경이 중요하므로 기존 ≈5초 컷·cover-fill 유지(인터리브 미적용).

14. **★해양생물 영상 자동 발굴 규칙(운영자 확정 · 소스 확대 · 풀 자동보충).** 제작 가능한 종이

      통과한 클립만 채택. 추가로 **조류·곤충·육상생물 배제**(`_EXCLUDE`)와 **연구·사체·해부·양식 클립
      배제**(`_BADCLIP`)로 채널 주제·화면 품질을 지킨다.
    - **★카테고리별 발굴(핵심 · 확정)**: 카테고리마다 **고유 검색어·분류 게이트**를 쓴다(`discovery._CATALOG`).
      과거 결함: 모든 생물 카테고리가 심해 검색어를 공유해 **미세조류를 검색조차 안 하고**(거미·해삼·문어가
      샘), 같은 종이 여러 카테고리에 중복돼 미리보기가 겹쳤다. 수정 내용 —
      - `deep_sea`=심해동물어, `marine_life`=일반 해양동물어, `marine_algae`=**규조류·와편모조류·식물성
        플랑크톤어**(`diatom`·`microalgae`·`phytoplankton`…). 각기 다른 검색어 집합.
      - **미세조류 게이트 반전**: 동물 배제(`_ANIMAL`) + 조류 **양성 확인**(`_ALGAE`, 학명·영문명·위키에
        조류 단서 필수). 동물 카테고리의 'plant 배제'는 미세조류엔 적용하지 않는다.

      2. **Tier3 배이름 레지스트리** — `Category:Shipwrecks by ship name`(171개+) 순회 → 실존 명명 난파선 +
         위키 사실(`_wiki_intro_by_name`). 구조화·중복 없는 무한 대상.
      3. **Tier2 사진→켄번즈** — 부족분은 고해상 난파선 사진을 후보로(`media_kind=photo`, `image_url`).
         제작 시 `footage.fetch_footage`가 `_kenburns_clip`으로 **16:9 느린 푸시인 영상**을 합성(정지 배가
         정적 피사체라 자연스럽고, 모션이 있어 정지-게이트 통과). 사진은 무한이라 공급이 사실상 끝없다.
      - 사진 후보 연결부: 후보→`promote_candidate`가 footage에 `media_kind=photo`·`image_url` 저장 →
        `_merge_discovered_seeds`가 _SEED로 전달 → `fetch_footage`가 켄번즈 분기. 썸네일·검증도 이미지 대응
        (`make_thumbnail`·`validate_source_url`은 사진이면 정지-게이트 미적용·고해상만 확인).
    - **영속화**: 발굴 결과는 `src/categories/*/discovered.json`(커밋됨)에 누적돼 회차 간 유지·중복 방지.
      import 시 `footage._SEED`·`data.SPECIES`에 병합된다. 워크플로 커밋 스텝에 `discovered.json` 포함.
    - **연결 지점(운영자 확정: 수동 소싱)**: 아침 자동 스케줄·제작 중 자동보충은 폐지. 관리자 페이지
      "소싱하기" 버튼(`source-species.yml`)이 후보를 발굴→`<cat>_candidates.json`에 담고, 운영자가
      썸네일·정보를 검토·선택하면 `generate-short.yml`이 `discovery.promote_candidate`로 풀 승격 후 제작.
      침몰선도 포함(학명 없어 배 이름·선종·수심 추출 + 결정론 일본어 카피). 최종 업로드는 운영자 확인.

15. **★코딩 후 실동작·오류 검증 규칙(절대 위반 금지 · "됐다"를 말하기 전 필수).** 코드를 고친 뒤에는
    **바뀐 경로가 실제로 동작하는지 끝까지 확인**하고 나서 보고한다. 단위 테스트 통과만으로 "완료"라고
    말하지 않는다(이 규칙이 없어 저장 시 옛 영상·소싱 404 같은 사고가 났다).
    - **① 최소 검증(항상)**: `import` 성공 + 관련 `pytest` + 문법검사(JS는 `node --check`, YAML은 파싱).
    - **② 실경로 검증(핵심)**: 바꾼 기능을 **실제로 한 번 굴려 본다**. 예) 파이프라인 변경 → 로컬 제작/부분
      실행, 소싱 로직 → 라이브 1회 발굴, 대시보드/워크플로 → **디스패치·배포 전제조건까지** 확인.
    - **③ 배포·등록 전제조건 체크(놓치기 쉬움 · 사고 사례)**:
      - **디스패치 워크플로는 기본 브랜치(main)에도 있어야** API로 실행된다(없으면 404) → 새 워크플로는
        main에도 등록했는지 확인.
      - **대시보드(worker) 변경은 배포되어야** 반영된다(`worker/**` 푸시 시 자동배포) → 배포 경로 확인.
      - **캐시**: 재생성 시 URL 고정이면 옛 캐시가 나온다 → 버전 파라미터 등 무효화 확인.
    - **④ 정직 보고**: 검증한 것과 **못한 것(예: 배포/키 없어 라이브 미확인)을 구분해** 명확히 적는다.
      증거 없이 "정상 작동"이라 단정하지 않는다. 실패·스킵은 그대로 보고(과장 금지).
    - 스킬 연동: `/shortgenerator` §3(출력 검증)·§9(릴리스 후 점검)와 함께 적용.
