# 보령 가을愛 스탬프 투어 웹앱

가을철 보령시 방문 활성화를 위한 **위치기반 스탬프 투어** 웹앱입니다.
관광지 3개소(각 반경 200m)에 도착해 방문 인증을 하고, 3곳을 모두 완주하면
자동 응모되어 **선착순**으로 보령지역사랑상품권을 지급합니다.

앱 설치 없이 **브라우저**에서 동작하며, 다른 프로젝트와 섞이지 않도록 저장소 내
전용 폴더(`boryeong-stamp-tour/`)에서 개발합니다.

## 현재 단계 — P1 프로토타입(프런트엔드)

동작하는 화면 흐름을 확인할 수 있는 프런트엔드 프로토타입입니다.

- 개인정보 동의 + 이름/휴대폰 입력
- Leaflet + OpenStreetMap 지도에 3개 관광지 표시(반경 원)
- GPS 위치로 반경 200m 방문 인증(Haversine 거리 계산)
- 3/3 완주 시 자동 응모 → 완료 화면(응모번호·선착순 순번)
- 재접속 시 이름+휴대폰으로 응모 내역 조회

> ⚠️ **프로토타입에서는 실제 개인정보를 서버에 저장하지 않습니다.**
> 상태는 브라우저 `localStorage`에만 mock 으로 저장됩니다.
> 실제 암호화 저장·선착순·조회 API는 **P2(Cloudflare Workers)** 에서 연동합니다.

## 실행 방법 (로컬)

```
cd boryeong-stamp-tour
python3 -m http.server 8000
```

브라우저에서 접속:
- PC: http://localhost:8000
- 휴대폰(같은 Wi-Fi): `http://<PC_IP>:8000`

> 위치 기능은 보안상 **HTTPS 또는 localhost** 에서만 동작합니다.
> 실제 휴대폰 현장 테스트는 HTTPS 배포 후 진행하세요.

### 테스트 모드
현장에 가지 않고 흐름을 확인하려면 지도 화면 하단의 **"테스트 모드"** 버튼을 켜세요.
켜져 있으면 인증 시 해당 지점 근처 좌표로 시뮬레이션합니다.
**배포 전 반드시 OFF(기본값)** 로 두고, 코드에서 제거하거나 비활성화하세요.

## 파일 구조

```
boryeong-stamp-tour/
├── index.html        # 진입점 (Leaflet CDN + 모듈 로드)
├── css/style.css     # 디자인 토큰(가을 팔레트) + 모바일 세로 UI
├── js/
│   ├── app.js        # 화면 전환/흐름(동의→지도→인증→완료→조회)
│   ├── geo.js        # Geolocation + Haversine 200m 판정
│   ├── spots.js      # 관광지 3개소 데이터(플레이스홀더 좌표)
│   └── api.js        # 백엔드 래퍼(현재 mock, P2에서 Workers 연동)
├── PRIVACY.md        # 개인정보 동의서/처리방침 초안
└── README.md
```

## 확정이 필요한 항목 (열린 사항)

- **관광지 3개소**: 명칭 + 정확한 위경도 (`js/spots.js`의 좌표는 플레이스홀더)
- **행사 기간**: `js/spots.js`의 `EVENT.periodText`
- **선착순 인원 상한 / 상품권 예산**: `EVENT.firstComeLimit`
- **발주·운영 주체 / 대행업체(수탁자)**: `PRIVACY.md`

## 다음 단계 (P2~) — 보안 백엔드

기존 인프라를 재활용하여 신규 의존성을 최소화합니다.

- **Cloudflare Workers + D1** (저장소 내 `book-carousel` Workers 패턴 재사용)
  - 개인정보 **AES-GCM 암호화 저장**, 키는 Workers Secret
  - 휴대폰 **솔트 해시**로 1인 1응모 중복 차단
  - **서버측 좌표 재검증**, 요청 rate limit
  - **선착순 순번**(서버 타임스탬프), 응모완료 조회 API
- **Telegram 봇**(기존 시크릿)으로 관리자에게 신규 응모 알림 — **PII 미포함**
- **관리자 페이지**: 토큰 인증 + 접근 로그, 암호화 내보내기
- 보유기간 경과 시 **자동 파기**

보안 설계 상세는 리포지토리 상위 계획 문서와 `PRIVACY.md`를 참고하세요.
