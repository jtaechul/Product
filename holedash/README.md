# HoleDash · 통과! 구멍 벽 🕳️

웹캠으로 내 몸을 인식해, 다가오는 **구멍 뚫린 벽**에 자세를 맞춰 통과하는 자세 매칭 게임.
선·도형·색만으로 그린 심플 그래픽 · 브라우저에서 바로 실행 · 에셋 0개.

## 어떻게 즐기나요

1. **시작하기** → 이름 등록 → **신체 등록**(차렷 → 양팔 벌리기)
2. 안쪽에서 구멍 벽이 점점 커지며 다가와요. 위쪽 안내(예: "🤸 양팔 쫙! T자")대로 몸을 만들어요.
3. 벽이 화면을 꽉 채우는 순간 **판정** → 깨끗하게 통과할수록 높은 등급(PERFECT~CRASH)과 점수.
4. 라이프 3개. 충돌(CRASH)하면 라이프 1개 감소. 15벽을 완주하거나 라이프가 다하면 종료.
5. 여러 명이 순차로 도전 → "그만하고 시상식"으로 최종 순위를 봐요.

## 실행 방법

브라우저에서 카메라를 쓰려면 **localhost 또는 https**가 필요해요.

```bash
# 이 폴더(holedash/)의 상위에서
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000/holedash/ 접속
```

배포본: `https://jtaechul.github.io/Product/holedash/`

> 💡 **카메라 없이 미리보기**: 주소 끝에 `?synth`를 붙이면(`…/holedash/?synth`)
> 웹캠·모델 없이 합성 캐릭터로 화면·흐름을 볼 수 있어요(개발/시연용).

## 구조 (3레이어 분리)

```
holedash/
├── index.html          화면(타이틀/등록/캘리브/결과/시상식) + 캔버스
├── style.css           디자인 토큰 + 화면/HUD/연출 스타일
├── vendor/             MediaPipe 자세 인식(로컬 동봉 → CDN 불필요·오프라인 OK)
│   ├── vision_bundle.mjs
│   ├── wasm/…
│   └── pose_landmarker_lite.task
└── src/
    ├── main.js         상태 머신 + 게임 로직(흐름·점수·판정 호출)
    ├── pose.js         [인식] 웹캠 + MediaPipe → 33관절 좌표
    ├── config.js       [데이터] 벽·등급·콤보·포즈 안내 등 수치
    ├── walls.js        [로직] 구멍 모양 그리기 + 충돌용 벽 마스크 생성
    ├── collision.js    [로직] 방법 C — 스켈레톤/벽 마스크 픽셀 비교로 충돌률
    ├── render.js       [연출] 카메라(거울)·스켈레톤·다가오는 벽·파티클
    └── audio.js        [연출] Web Audio 절차적 효과음(에셋 0개)
```

## 핵심 설계 메모

- **거리 무관 공정성**: 구멍 크기를 매 프레임 플레이어의 **현재 어깨너비**에 비례시켜,
  앞뒤로 움직여도 일관되게 판정.
- **0점이 잘 안 나오게**: GOOD·OUCH로도 점수를 쌓아 저학년도 끝까지 재미.
  고학년은 PERFECT·콤보로 고득점 → 실력차 반영.
- **그래픽 교체 용이**: 인식/로직과 렌더가 분리돼, 나중에 스켈레톤·벽을 스프라이트로 갈아끼우기 쉬움.

## 구현 현황 (기획서 로드맵 대비)

- ✅ 1단계: 웹캠+MediaPipe, 벽 접근, 충돌률·등급·점수
- ✅ 2단계: 연속 벽·난이도 상승, 캘리브레이션, 콤보·라이프
- ✅ 3단계: 통과/충돌 연출·화면 흔들림·효과음, 충돌 판정 방법 C(마스크)
- ✅ 4단계: 순차 플레이·최종 순위/시상식
- 🔜 5단계: 변형 벽 확장(회전·이동은 일부 포함), 엔드리스 모드, 그래픽 에셋 교체
