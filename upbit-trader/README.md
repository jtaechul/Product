# Upbit 자동매매 (Upbit Auto Trader)

Upbit Open API를 활용한 가상화폐 자동매매 프로젝트입니다.
**안전 우선** 원칙으로, 돈을 거는 단계는 가장 마지막에 추가합니다.

> ⚠️ **경고**: 자동매매는 원금 손실 위험이 있습니다. 이 코드는 교육/연구 목적의
> 출발점이며, 실거래에 사용하기 전 충분한 백테스트와 소액 검증을 거치세요.
> 작성자는 이 코드 사용으로 인한 어떠한 손실에도 책임지지 않습니다.

## 개발 로드맵

- [x] **1단계: 시세 조회** — 인증 없이 마켓/캔들/현재가/호가 가져오기
- [x] **2단계: 전략** — 이동평균 교차 / RSI / 변동성 돌파 / 볼린저밴드 / MACD (5종)
- [x] **3단계: 백테스트** — 과거 데이터로 전략 검증 (+파라미터 최적화, 학습/검증 분할)
- [x] **4단계: 모의매매** — 실주문 없이 가상 계좌로 거래 일지 기록
- [x] **리스크 관리** — 손절/익절(stop-loss/take-profit)
- [x] **대시보드** — 결과를 차트 HTML 로 시각화
- [x] **5단계: 실거래 봇** — JWT 인증 + 시장가 주문, dry-run 기본 + 투자금 상한
- [x] **잠수함 급등 매매 v1 (1분봉)** — 스캐너 + 추적기 + 자동봇
  - ⚠️ **독립 표본 검증 실패**: out-of-sample -756%(t=-7.89). 실거래 금지(교육용).
- [x] **잠수함 급등 매매 v2 (스윙)** — 시간봉, 빠른 진입/인내 청산 + 추세 게이트
  - ✅ **독립표본 3종 검증 통과**: Bitfinex +1400%(t=4.47), 일봉 +1520%(t=3.74)
- [ ] 6단계: 실시간 모니터링/알림 (텔레그램 등)

## 잠수함 알트코인 급등 매매

조용히 가라앉아(축적) 있다가 거래량이 터지며 급등하는 알트코인을 포착해
빠르게 진입하고, 트레일링 스탑으로 빠르게 청산하는 별도 시스템입니다.

> ## 🛑 독립 표본 검증 결과: 이 1분봉 전략은 실거래에 쓰지 마세요
>
> Binance 2024(5코인·43거래)에서 보였던 +5.7%는 **과최적화/행운**이었습니다.
> 완전 독립 표본(**Bitfinex 2013~2019, 7코인, 1,720거래**)에 같은 전략을
> 재생하니 **누적 -756%, 승률 24%, t=-7.89 — 7코인·7연도 전부 손실**.
> 파라미터를 어떻게 바꿔도(트레일 8%, 조기손절 제거, 익절 고정) 전부 마이너스
> (-261%~-336%). → `python -m scripts.backtest_bitfinex` 로 재현 가능.
>
> **무엇이 진짜고 무엇이 가짜인가:**
> - ✅ **일봉 스케일의 엣지는 진짜** (228 코인-년, t=6.5): "거래량 급증 돌파를
>   사서 며칠~수주 추세를 트레일링" — 이건 통계적으로 유의합니다.
> - ❌ **1분봉 스캘핑 구현은 가짜**: 일봉 엣지가 1분봉으로 내려오면 1분 노이즈와
>   비용(수수료+슬리피지)에 잘게 썰려 사라집니다. 조기손절이 출혈의 주범.
>
> **권장 방향**: (1) 시간프레임을 **시간봉/일봉**으로 올려 엣지가 살아있는 곳에서
> 매매하거나, (2) 1분봉을 고집한다면 진입 빈도를 크게 줄이고(훨씬 높은 신호 문턱)
> 비용을 낮춰야 합니다. 아래 1분봉 봇/스캐너 코드는 **교육·실험용**으로만 두세요.

## ✅ 스윙 전략 (재설계 — 독립표본 3종 검증 통과)

위 실패를 받아들여 **"빠른 진입, 인내하는 청산"**으로 다시 짰습니다. 실시간 1분봉으로
돌파를 즉시 포착하되, 보유·청산은 **시간봉 스케일**(넓은 트레일링, 수시간~수일)이라
1분 노이즈·비용에 썰리지 않습니다. 추세 게이트(BTC·코인 자기 MA 위)로 약세장 차단.

**같은 파라미터로 3개 독립 데이터셋 검증:**

| 데이터셋 (거래소·기간·통화) | 거래 | 누적 | PF | t | 유의성 |
|------|-----|------|-----|---|--------|
| Bitfinex 2013~2019 USD (독립) | 454 | +1400% | 1.79 | **4.47** | ✓ |
| Coin Metrics 일봉 2010~2026 | 75 | +1520% | 5.78 | **3.74** | ✓ |
| Binance 2024~2025 USDT | 32 | +87% | 1.66 | 1.18 | 분포 일치 |

- 1분봉 버전이 **-756%(t=-7.89)**였던 바로 그 Bitfinex 표본에서 **+1400%(t=+4.47)**.
- 7코인 전부·6/7연도 흑자. 파라미터를 흔들어도(트레일 12~20%, 손절 8~10%,
  거래량배수 2.5~4, BTC게이트 on/off, 4시간봉) **전부 t>4 유지** — 칼날 위가 아님.
- 승률 ~45%, 평균 수익(승) 2~3.5배 × 평균 손실 — 추세를 끝까지 먹는 구조.

```bash
# 백테스트(교차검증): 데이터셋만 바꿔 같은 전략 재생
python -m scripts.backtest_swing --source bitfinex --tf 1h   # 독립 검증
python -m scripts.backtest_swing --source daily    --tf 1D
python -m scripts.backtest_swing --source binance  --tf 1h --show

# 실시간 매매 봇 (모의 기본 / --live 시 소액 실거래)
python -m scripts.swing_trade --max-positions 3 --invest 100000
python -m scripts.swing_trade --invest 50000 --live

# 검증용 Bitfinex 1분봉 데이터 재생성(거래소 API 막혀도 GitHub로 다운로드)
bash scripts/fetch_bitfinex.sh
```

> 검증은 강하지만 **여전히 위험**합니다: 백테스트는 체결·슬리피지를 단순화하고,
> 미래 시장이 과거와 같다는 보장은 없습니다. 반드시 모의로 충분히 돌린 뒤
> 소액으로만 실거래하세요.

```bash
# 1) 스캐너: 전체 KRW 마켓을 5분봉으로 훑어 급등 후보 순위표 (조회만, 안전)
python -m scripts.scan_pumps --watch 60

# 2) 추적기: 포착한 코인 1개를 1분봉/현재가로 정밀 추적 매매 (모의 기본)
python -m scripts.track_coin --market KRW-XYZ

# 3) 통합 봇: 스캐너(백그라운드) + 다종목 추적을 하나로 (모의 기본)
python -m scripts.auto_trade --max-positions 3 --invest 6000
python -m scripts.auto_trade --invest 6000 --live   # 소액 실거래

# 4) 백테스트: 합성 멀티코인(진짜/가짜 펌프 혼합)에 전략 재생 → 성적
python -m scripts.backtest_pumps --coins 36 --days 4
python -m scripts.backtest_pumps --trail 2.5 --stop 4 --arm 1.5  # 파라미터 실험

# 5) 실데이터 백테스트: data/ 에 Binance 월별 1분봉 zip 을 넣고 실행
#    (예: NEOUSDT-1m-2024-11.zip — https://data.binance.vision 에서 다운로드)
python -m scripts.backtest_real --start 2024-10-01 --end 2024-12-31
python -m scripts.backtest_real --show-trades --dump-trades trades.csv
```

**실데이터 검증 (Binance 1분봉, 2024-10~12, ADA/LINK/MANA/NEO/TRX)**:

| 설정 | 거래 | 승률 | 누적 | PF |
|------|-----|------|------|-----|
| 보완 전 | 55회 | 29% | -10.6% | 0.81 |
| **보완 후 (현재 기본값)** | 43회 | 35% | **+5.7%** | **1.14** |

보완 내용 (실데이터로 검증):
- **추격 차단(max_momentum +4%)**: 15분에 이미 4% 넘게 오른 코인은 진입 금지.
  스파이크 꼭대기 매수가 손실의 주범이었음 — 이 필터 하나로 흑자 전환.
- **초기 손절(initial_stop -2.5%)**: 트레일링 활성화 전에는 -5% 대신 -2.5%
  손절. 기대수익 중립이지만 1거래 최대 손실을 절반으로 제한.
- **잠수함 게이트(min_dormancy)는 기본 끔**: 직전에 조용했던 코인만 골랐더니
  큰 펌프(이미 움직이기 시작한 코인)까지 걸러내 성적 악화. 옵션으로만 제공.
- **코인별 주간 브레이크(기본 켬)**: 한 코인의 최근 7일 누적 손익이 -4% 이하면
  3일간 그 코인 진입 금지. 고변동 장세의 '하루 -2%씩 새는' 느린 출혈을 차단
  (ZEC 2025-09~10: -19.7% → -11.8%, 2024 수익에는 무영향).
  일일 손실 한도는 이 패턴에 안 걸려서(하루 1거래꼴) 효과가 없었습니다.

⚠️ **검증의 한계 (정직 고지)**:
- +5.7%는 상위 3개 거래가 만든 수익입니다(상위 1개 제외 시 -4.6%).
  펌프 추격 전략의 본질(소수 대박이 다수 소손실을 부양)이지만,
  43거래 표본으로는 통계적으로 유의하지 않습니다(t≈0.3).
- ZEC 같은 지속 고변동 장세는 브레이크로 완화될 뿐 여전히 손실입니다.

### 대규모 표본 검증 (일봉, 228 코인-년)

1분봉 데이터를 더 못 구하는 환경에서, **핵심 엣지가 우연인지**를 따로 검증했습니다.
Coin Metrics 공개 일봉(24개 코인, 2010~2026)으로 동일 구조(거래량 급증 돌파 매수
+ 트레일링/손절 청산)를 재현:

```bash
python -m scripts.backtest_daily              # 전체 통합 + 연도별 워크포워드
python -m scripts.backtest_daily --vol-surge 4 --trail 0.15   # 파라미터 흔들기
```

| 지표 | 결과 |
|------|------|
| 거래수 | 491회 (24개 코인) |
| 평균손익/거래 | +14.7% |
| 손익비(PF) | 4.0 |
| 평균수익 95% 신뢰구간 | **[+10.4%, +19.7%], t=6.5 → 통계적 유의** |
| 흑자 코인 | **24/24 (100%)** |
| 흑자 연도 | 13/16년 (손실: 2018·2022·2025 — 모두 약세장) |

파라미터를 흔들어도(vol_surge 2.5~4, trail 12~15%, stop 8~10%) 모두 t>5로 유의.
→ **핵심 엣지(거래량 급증 돌파 + 트레일링)는 일봉 스케일에서 강하게 일반화됩니다.**

다만 정직하게:
- 이건 **일봉 프록시**입니다. 1분봉 실거래 시스템 그 자체(슬리피지·체결·5분
  스캔 주기)는 아니며, 1분봉 표본은 여전히 43거래뿐입니다.
- 추격 차단 필터는 **시간프레임 의존적**입니다: 1분봉에선 흑자 전환에 결정적이었지만,
  일봉에선 오히려 약간 손해(+16.2%→+14.7%) — 일봉에선 15% 움직임 뒤에도 추세가
  이어지기 때문. 그래서 일봉 검증은 '엣지의 존재'를 입증할 뿐, 1분봉 파라미터의
  최적성까지 보장하진 않습니다.
- **약세장에서는 일봉에서도 손실** — ZEC 결과와 일치.

#### 추세 필터로 약세장 방어 (검증 완료)

```bash
python -m scripts.backtest_daily --btc-ma 200 --self-ma 100   # 추천 조합
```

| 설정 | 거래 | 평균/거래 | PF | 2018 | 2022 | 2025 |
|------|-----|----------|-----|------|------|------|
| 추세필터 없음 | 491 | +14.7% | 4.0 | **-114%** | **-158%** | -81% |
| BTC200 + 자체100MA | 358 | +15.3% | **4.3** | **-11%** | (진입 0) | -61% |

- **고전적 약세장(2018·2022)은 추세 필터가 거의 제거**합니다 — 이때는 BTC도
  200일선 아래라 게이트가 신규 진입을 막습니다. PF도 4.0→4.3으로 개선.
- **단, 2025년은 못 막습니다**: BTC는 73% 기간 200일선 위(강세처럼 보임)였는데
  알트코인만 무너진 장세 — 24거래 중 12개 손절, 특정 코인이 아닌 전반적 약세라
  BTC 기준 필터로는 거를 수 없습니다. 이 잔여 손실은 **손절(거래당 평균 -10%)로
  통제**될 뿐, 진입 단계에서 예방되지 않습니다.
- 결론: 추세 필터는 시장 전체 약세는 막지만 '알트만 약한' 국면은 못 막습니다.
  실거래에선 추세 필터 + 손절 + 주간 브레이크를 함께 써야 합니다.

`data/download_links.txt` 의 1분봉 검증 세트를 받아주시면 1분봉 일반화도 확인합니다.
**그 전까지 실거래 투입은 권장하지 않습니다.**

**조기손절(early-cut)**: 진입 후 5분 내 상승 모멘텀이 살아나지 못하면 작은
손실로 빠르게 탈출 — 가짜 펌프(불 트랩)에 물렸을 때 손절폭까지 버티지 않습니다.

**수익↑/손실↓ 장치**: 초기 손절 + 손절 하한 + 트레일링 스탑 + 본전 스탑(수익권
반납 차단) + 단계별 트레일링(수익 클수록 좁게) + 일일 손실 한도 + 재진입 쿨다운.
보유 코인은 캔들이 아닌 **현재가 일괄 조회**로 빠르게(기본 2초) 감시합니다.

> ⚠️ 급등 추격은 가장 위험합니다. 신호가 잡힐 땐 이미 오른 경우가 많고,
> 잠수함 코인은 거래량이 적어 슬리피지가 큽니다. 반드시 모의로 충분히
> 검증한 뒤 소액으로만 실거래하세요.

## 주요 스크립트

```bash
python scripts/fetch_market_data.py KRW-BTC   # 시세 조회 (네트워크 필요)
python scripts/check_balance.py               # 내 잔고 (API 키 + IP 등록 필요)
python scripts/run_backtest.py                # 전략 성적 비교
python scripts/run_backtest.py --stop-loss 0.05 --take-profit 0.15  # 손절/익절 적용
python scripts/optimize.py                    # 전략별 좋은 파라미터 탐색
python scripts/validate.py                    # 학습/검증 분할로 과최적화 점검
python scripts/paper_trade.py --strategy bb   # 모의매매 거래 일지
python scripts/make_dashboard.py              # dashboard.html 생성 → 브라우저로 열기
python -m streamlit run app/dashboard_app.py  # 실시간 웹 대시보드
python -m scripts.live_trade --strategy vb              # 자동매매 봇 (모의/dry-run)
python -m scripts.live_trade --strategy vb --max-invest 6000 --live  # 실거래(소액)
```

위 백테스트/최적화/검증/모의매매/대시보드는 **API 키 없이** 합성 데이터로 동작하며,
`--csv 파일.csv` 로 실제 OHLCV 데이터를 넣으면 그대로 실데이터 검증이 됩니다.

## Upbit Open API 메모

- 시세 API: 인증 불필요. `https://api.upbit.com/v1/...`
- 거래 API: JWT 인증 필요 (Access Key / Secret Key). 5단계에서 추가 예정.
- API 키 발급: **PC 웹(upbit.com) 전용** (모바일 앱에는 메뉴 없음). 로그인 →
  마이페이지 → Open API 관리(`https://upbit.com/mypage/open_api_management`) →
  권한 선택 → **허용 IP 등록** → 본인 인증 → Access/Secret key 발급.
  - 사전 조건: **고객확인(KYC) + 2채널 인증(2FA)** 완료해야 발급 가능.
  - **Secret key 는 최초 발급 화면에서 단 한 번만** 표시되므로 즉시 안전하게 보관.
- 자동매매에는 **자산조회 + 주문** 권한만. **출금 권한은 절대 켜지 마세요.**
- 호출 횟수 제한(rate limit)이 있으니 과도한 요청은 피하세요.
- 공식 문서: https://docs.upbit.com/
