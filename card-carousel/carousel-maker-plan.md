## 1. 프로젝트 개요

### 1-1. 목적
- 이미지 1장(또는 여러 장)을 입력하면 다국어 카드뉴스(캐러셀)를 자동 생성하는 웹 프로그램 개발
- 인스타그램 보너스 프로그램 수익화 파이프라인의 "재가공~출력" 구간 자동화
- 광고 단가가 높은 미국·일본 유저 타겟 다국어 계정 운영 지원

### 1-2. 설계 원칙
- **소스 중립형(source-agnostic)**: 이미지 출처와 무관하게 사용자가 업로드한 이미지를 처리
- 라이선스 판단·소싱 책임은 사용자에게 있으며, 프로그램은 재가공 도구 역할만 수행
  (STEP 0 자동 추적은 **판단 보조 수단**이며 최종 책임을 대신하지 않음)
- 기존 북 캐럿셀 프로그램의 검증된 골격(생성→검증→캡션→저장·복사)을 재활용하는 **개조 프로젝트**

### 1-3. 지원 언어
- 한국어, 영어, 일본어 (3개 언어)
- 중국어 제외 (중국 내 인스타그램 차단으로 타겟 유저 부재)
- 단순 번역이 아닌 현지화(localization): 언어별 관용 표현·해시태그 별도 생성

## 2. 전체 워크플로우

### STEP 0. 원본 추적 [프로그램 + Gemini API] — v1.1에서 자동화 (구현 완료)
- 수행 주체: 프로그램 (`/api/trace-origin` + 입력부 "원본 추적" 버튼)
- 방법: 업로드한 이미지를 **Gemini API(이미지 인식 + 구글 검색 그라운딩)** 로 분석
  - 이미지 속 단서 추출: 작품명·인물·워터마크·로고·서명·화면 속 텍스트
  - 구글 검색 그라운딩으로 추정 원본 출처(작품/작가/플랫폼) + 근거 URL 목록 제시
  - **저작권 위험도 자동 판정**: high(상업 콘텐츠 무단 재가공 위험) / medium(확인 필요) / low(퍼블릭 도메인·CC 가능성) / unknown
- 전송 최적화: 원본은 카드 배경에 쓰고, API에는 긴 변 1024px 축소본(JPEG)만 전송
- **방식의 한계(중요)**: 구글 렌즈식 "픽셀 단위 완전 일치" 역검색이 아니라 내용 인식 기반 **추정**.
  결과 카드에 구글 렌즈 수동 확인 안내를 함께 표시하며, 최종 라이선스 판단은 사용자 몫
- **원본 후보 이미지 교체 (v1.2)**: 추적된 출처 페이지(최대 3곳)의 대표 이미지(og:image)를
  추출해 후보 썸네일로 표시 → 클릭 한 번으로 업로드 이미지를 원본(보통 고화질)으로 교체.
  정확도가 더 필요하면 구글 Vision 역검색(Web Detection, 유료)을 추후 추가 가능(B안 예비)
- 비고: 유료 SerpApi(렌즈 역검색) 안은 비용 문제로 제외 유지 → 이미 결제한 Gemini로 대체

### STEP 1. 입력 [프로그램] — Phase 1 구현 완료
- 이미지 업로드: 1장 또는 여러 장 (드래그 앤 드롭 + 클릭 선택, 썸네일 미리보기·삭제)
- 모드 선택 (구현 — Phase 4): 출력 버튼으로 분기 — [이미지 전체 저장]=게시물(1:1/4:5 프리셋 토글),
  [릴스 세트]=릴스(9:16 고정). 하나의 결과물에서 두 형식 모두 출력 가능
- 주제 입력 방식 선택:
  - 모드 A (수동, 구현 완료): 주제·핵심 메시지 직접 입력
  - 모드 B (AI 자동, 구현 완료 — Phase 3): **Gemini vision**이 업로드 이미지(최대 3장)를 분석 →
    주제 1개 자동 입력 + 핵심 메시지 후보 3개를 칩으로 제안 → 사용자가 선택·수정 (`/api/suggest-topic`)

### STEP 2. 콘텐츠 생성 [프로그램 + AI API] — v1.2에서 언어 온디맨드로 개선
- Claude API가 주제 기반 슬라이드별 문구 작성
- 구조: 후킹 문구(1장) → 본문(중간 슬라이드) → 마무리 CTA(팔로우·저장 유도)
- **언어 온디맨드 (v1.2)**: 입력부에서 선택한 언어 1개로만 생성(비싼 main 모델).
  다른 언어는 탭을 처음 누를 때 `/api/translate`가 **싼 모델(light)로 번역·현지화**해 캐시.
  → 한 언어만 쓰는 날은 3개 언어 생성 대비 약 1/3 비용, 3개 언어를 다 써도 번역이
  싼 모델이라 이전보다 저렴. 캡션·릴스 대본도 함께 번역됨
- 현지화 원칙 유지: 직역 금지, 해시태그는 언어권 실제 태그로, 일본어 정중체(です・ます)
- "다시 생성" 시 원문이 바뀌므로 기존 번역 캐시는 무효화(현재 언어만 유지)

### STEP 3. 품질 검증 [프로그램 + AI API] — Phase 3 구현 완료
- 기존 북 캐럿셀의 품질 검증 루프 재활용 (최대 2회 자동 검증·재생성)
- 검증 항목 (구현): 주제 부합·사실 정확성·안전성·공감 유도·문장 품질 +
  **3개 언어 세트를 함께 검토** — 오역, 기계번역투·어색한 표현(특히 일본어) 감점

### STEP 4. 렌더링 (텍스트 오버레이) [프로그램] — Phase 2 구현 완료 (게시물 카드 기준)
- 위치 자동 배치 (구현): 이미지를 3x3 그리드로 분석해 상단/하단 밴드의 **복잡도**(경계선 밀도 +
  명암 편차)를 비교 → 덜 복잡한(피사체가 없을 가능성이 높은) 쪽에 텍스트 배치.
  동률이면 하단(인스타 관례). 미리보기(DOM)와 저장본(캔버스 1080px)에 동일 적용
- 색상 자동 결정 (구현): 텍스트가 놓일 밴드의 평균 밝기 분석 →
  밝은 배경 = 어두운 글씨 + 흰 번짐(halo) + 흰 스크림 / 어두운 배경 = 흰 글씨 + 검은 스크림.
  페이지 번호(좌상단)는 상단 밴드 밝기 기준으로 별도 결정
- 크기 자동 조절 (구현): 글자 수 기반 autoFitLines(기존 엔진 재활용).
  언어별 최적 폰트 분리 적용(한글/일문/영문)은 Phase 3(다국어)에서
- 릴스 모드: 상하단 UI 겹침 영역 회피 배치는 기존 안전 밴드(200~1430px) 유지,
  프리셋 분기는 Phase 4에서

### STEP 5. 출력 [프로그램] — 언어 탭 구현 완료 (Phase 3 → v1.2 온디맨드로 개선)
- 언어 탭(한/영/일)별 미리보기 (탭 전환 시 카드 5장·캡션·폰트가 해당 언어로 전환.
  v1.2: 번역 안 된 탭은 흐리게 표시, 첫 클릭 시 지연 번역 후 캐시)
- [이미지 전체 저장] 버튼: 현재 선택된 언어의 카드뉴스 일괄 다운로드 (언어별 폰트 적용)
- [캡션 복사] 버튼: 현재 언어 캡션+해시태그 원클릭 복사 (해시태그는 언어권별 현지화 태그)

## 5. 기술 스택 및 비용

### 5-1. 기술 스택
- 배포: Cloudflare Workers (기존 환경 그대로, worker명 `card-carousel`)
- AI: Claude API — 문구 생성, 번역·현지화, 주제 자동 추출, 품질 검증
- **AI: Gemini API — STEP 0 원본 추적 (이미지 인식 + 구글 검색 그라운딩)** ← v1.1 추가
- 렌더링: 브라우저 Canvas API — 텍스트 오버레이, 서버 비용 0원
- 폰트: 언어별 웹폰트 (한글/일문/영문 각 1종, 무료 라이선스 폰트 사용)

### 5-2. 비용 구조
- 유료: Claude API 호출 — v1.2 언어 온디맨드로 최적화: 생성은 선택 언어 1개만(main 모델),
  추가 언어는 지연 번역(light 모델, 약 1/10 단가)
- **유료: Gemini API 호출 (원본 추적 — 이미지당 1회, 결제 완료)** ← v1.1 추가
- 무료: 렌더링, 다운로드, 캡션 복사, Cloudflare Workers 무료 티어
- 제외된 비용: 역방향 검색 전용 API(SerpApi 등) — Gemini로 대체하여 계속 제외

### 5-3. 필요 시크릿 (Cloudflare Workers)
- `ANTHROPIC_API_KEY` — 문구 생성·검증·캡션
- `GEMINI_API_KEY` — 원본 추적 (v1.1 추가)
- `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` — 알림 (선택, Phase 5)
- ⚠️ 배포 전 KV 네임스페이스를 card-carousel 전용으로 새로 생성해 `wrangler.toml`의 id 교체 필요

## 7. 리스크 관리

### 7-1. 저작권
- 역방향 검색으로 원본을 찾아도 사용권이 생기는 것은 아님 (출처 확인 ≠ 사용 허락)
- **자동 원본 추적(Gemini)은 내용 인식 기반 "추정"** — 픽셀 일치 역검색이 아니므로
  오인 가능성이 있음. 위험도 high/medium 판정 이미지는 구글 렌즈로 직접 재확인 권장
- 안전 소싱 우선순위: ① 퍼블릭 도메인·CC0 ② CC-BY (출처표시) ③ 자체 생성(AI·직접 촬영)
- "무료로 보임 ≠ 무료로 사용 가능" — 파일 단위 라이선스 확인 필수

### 7-2. 플랫폼 정책

- 인스타 커뮤니티 가이드라인 준수

### 7-3. 수익화 조건 (참고)
- 프로페셔널 계정 + 최근 3개월 월간 총 조회수 100만 회 이상 → 보너스 탭 활성화
- 단가: 조회수 1만 회당 약 1달러 수준, 매월 정산
