# 쿠팡 어필리에이트 쇼츠 팩토리 — Claude Code 프로젝트 규칙

> 마스터 스펙(SSOT): 이 폴더의 `coupang-shorts-pipeline-spec.md` — 모든 구현은 스펙 §8 로드맵의
> Phase 순서를 따르고, 각 Phase의 DoD를 충족한 뒤에만 다음 Phase로 진행한다.

> ## ⭐⭐ 전면 개편 진행 중 (2026-07-14 사용자 확정 — 완전 수동·사용자 주도 파이프라인)
> 방향 전환: 자동 소재발굴·자동제작을 **폐지**하고, 6단계 **수동 협업** 파이프라인으로 재편한다.
> 1. **소싱=사용자**(상품 캡처+대표 이미지/영상+어필리에이트 링크 업로드) 2. **Claude=기획**(분석→
> 페인포인트→기획→스토리보드→대본→구간별 이미지/영상 제안) 3. **사용자=선택**(구간별 다중선택)
> 4. **Claude=제작**(선택본으로 Shorts+제목+설명+해시태그) 5. **사용자=검수**(항목별 '다시 생성'
> 버튼, OK면 '올리기') 6. **사용자=유튜브 업로드**.
> - **Part A 삭제 완료(사용자 확정)**: 소재 리서치 전체(`src/research/`, `shorts-research.yml`,
>   `config/competitors.yaml`, 관리자 '채널' 탭) · 자동제작 cron(`shorts-produce.yml` schedule) ·
>   `src/video/veo.py`(폐기 유료영상) · `src/phase0_spike.py`(초기 테스트). 워크플로 `phase0` 모드 제거.
> - **유지(사용자 확정)**: 음성 나레이션(TTS) · BGM · 밈 카드 · 텔레그램 알림 · 쿠팡 API 모듈(보존).
> - **B1 완료**: 기획 승인 단계. `pipeline --plan`이 기획(concept: 의도·타깃·후킹)+스토리보드(라인
>   `scene`)+대본을 `data/plans/{row_hash}.json`에 저장(워크플로 `plan` 모드가 커밋), 관리자 '기획' 탭에서
>   검토→승인('이미지 찾기')/다시생성. candidates·produce는 승인된 기획이 있으면 재생성 없이 사용.
> - **B2 완료**: 구간별 이미지 다중선택 → 슬라이드쇼. selections 포맷 `{lines:[{line_i,picks:[...]}]}`,
>   `load_selections`가 라인당 배열 반환, `render._plan_line_scenes`가 라인 구간을 균등 분할해 순서대로 재생.
>   관리자 이미지 탭 다중선택(순서 배지).
> - **B3 완료**: 제작하기 = `pipeline`(produce `--no-upload`) 렌더+릴리스만(자동 업로드 안 함).
>   release_meta에 `row_hash`(상품→영상 매칭).
> - **B4 완료**: 검수 + 항목별 다시생성. `pipeline --regen FIELD`(title|headline|description|hashtags)이
>   기획의 그 항목만 새 안 1개로 즉시 교체(`generate.regenerate_field`), 영상 다시 만들기=재제작.
> - **B5 완료**: 올리기 = `pipeline --upload`가 릴리스 shorts-run에서 영상 받아 유튜브 업로드
>   (`_find_and_download_video`+`youtube.upload`, 기획 제목·설명·해시태그 사용). **SHORTS_YT 인증 3종 필요**.
>   **업로드 기본 = 공개(public, 2026-07-17 사용자 확정)**: 검수는 올리기 전에 끝내는 흐름이므로 바로
>   공개 + 고지 댓글 무조건 자동 등록. (비공개(private)는 유튜브가 댓글 자체를 막아 403 — run129 원문
>   검증, 스코프 문제 아님·토큰 정상.) 댓글 실패 시 텔레그램 알림이 원인별 안내 + 복붙용 댓글 내용 포함
>   (`youtube.py` 힌트 분기). '고정' 지정만은 유튜브 API 미지원 → 앱에서 1탭. **비공개 잠금 감지**:
>   미심사 API 프로젝트의 공개 업로드를 유튜브가 private로 강제하면 업로드 코드가 요청/실제 공개범위
>   불일치를 감지해 텔레그램 경고(심사 폼 안내로 이어감).
> - **B6 완료**: 관리자 '제작' 탭을 제작·검수·올리기 스튜디오로 재구성 + 탭 순서 상품→기획→이미지→
>   제작→영상→설정로 흐름 통합(옛 자동제작·리서치 버튼 폐지). 워크플로 mode: plan·candidates·produce·
>   regen·upload(+no_upload·field 입력).
> - **차별점 3안 + 선택 게이트 (2026-07-17 사용자 확정)**: "재밌는데 뭘 파는지 모르겠다" 개선. plan이
>   상품 자료(PDF·캡처 notes)에서 **차별 페인포인트 + 해소 기능·구조 + 작동 방식**을 서로 다른 각도
>   3안으로 추출(`src/product/mechanism.py` → `data/mechanism/{hash}.json`, 워크플로 커밋).
>   **⭐ 게이트: 운영자가 방향을 확정(confirmed/chosen/custom)하기 전에는 plan이 대본을 만들지 않고
>   멈춘다**(3안 저장→텔레그램 알림→return 0). 관리자 '만들기' 탭의 **독립 메뉴 '② 기획 방향 선택'**
>   (등록과 이미지 선택 사이 · #dirView)에서 진행: 등록 후 **[기획방향 수립] 버튼**을 눌러야 3안 추출
>   시작(자동 시작 폐지) → 3안 도착 시 그 메뉴에 표시(pollPlan 감지, 미확정이면 기존 대본도 숨김) →
>   **선택/직접 의견(최우선)/3안 다시 뽑기** → '이 선택으로 기획·대본 만들기'가
>   확정 저장 후 plan 재실행(waitSha로 새 대본까지 대기). 확정안은 `product["차별점_작동방식"]`로
>   대본에 주입. 자료·키 없음/추출 실패는 게이트 없이 진행(운영자가 막히지 않게), produce/candidates는
>   비게이트(확정 없으면 1안 사용).
>   대본 아크도 개편(script_gen.md): ③=기존 같은 종류 제품들의 한계 저격(④ 셋업), ④=(a)남들이
>   못 한 것 (b)무슨 기능·구조 (c)어떻게 작동(숫자) 3답 필수, **④+⑤ 라인 4개 이상**, 자가검증
>   ("처음 본 사람이 무슨 제품·뭐가 다른지·어떻게 되는지 말할 수 있나") 추가.
> - **개편 전체 완료 — 남은 것**: (1) 작업 브랜치를 main에 반영해 라이브 전환 (2) 실동작 검증
>   (실상품으로 기획→이미지→제작→검수→업로드 CI 라운드트립) (3) 업로드용 SHORTS_YT 인증 3종 등록.
>   (아래 기존 문서 중 리서치·cron·자동화 서술은 폐지됨 — 참고만.)

## ⭐ 핵심 규칙 — 사용자 노코드 원칙 (항상 적용)

- 사용자는 **코드·설정·CSV·YAML 파일을 직접 입력하거나 편집하지 않는다.** 사용자 입력이 필요한
  모든 조작(상품 등록·삭제, 제작 실행, 벤치마크 채널 관리 등)은 **관리자 페이지 UI**로 제공한다:
  [shorts-admin.jtaechul.workers.dev](https://shorts-admin.jtaechul.workers.dev)
  (`admin/` 폴더, Cloudflare Workers 정적 에셋, `deploy-shorts-admin.yml`로 자동 배포).
- 새 기능이 사용자 입력을 요구하게 되면 **관리자 페이지에 해당 입력 UI를 함께 추가**한다.
  사용자에게 파일 편집·코드 수정을 안내하는 것은 규칙 위반.
  (허용 예외: GitHub 시크릿 등록, 외부 서비스 키 발급처럼 외부 사이트 자체 화면에서만 가능한 작업)

## ⭐ 핵심 규칙 — 비판적 파트너 원칙 (항상 적용)

- **사용자 의견을 무조건 반영하지 않는다.** 모든 지시·의견은 프로젝트의 궁극 목적
  (유튜브 쇼츠 → 쿠팡 어필리에이트 수익)에 비추어 비판적·객관적으로 검토하고,
  특히 **유튜브(구글) 알고리즘·검색 SEO·정책 리스크** 관점에서 문제가 보이면
  반영 전에 이견·대안·개선방안을 근거와 함께 제시한다.
- Claude 자신의 과거 결정도 같은 기준으로 재검토 대상이다. 알고리즘 관련 판단에는
  가능한 한 근거(정책명·지표)를 붙이고, 추측이면 추측이라고 밝힌다.

## ⭐ 핵심 규칙 — 유료 AI '영상' 생성 금지 (2026-07-13 사용자 확정 · 절대 위반 금지)

- **Gemini/Veo 등 유료 AI '영상' 생성 API를 절대 호출하지 않는다.** 실제 테스트 결과 품질이
  제작비 대비 값어치가 없다는 사용자 최종 결정. "다시는 제미나이 영상 API 건드리지 마."
  `src/video/veo.py`는 흔적만 남은 사용 안 하는 모듈 — **재연결·재도입 금지**.
- **단, Gemini '텍스트' 생성은 허용·사용한다(2026-07-13 사용자 지시)**: 대본(구어체가 Claude보다
  낫다는 판단, `script.provider=gemini`)과 이미지 검색어 생성에 Gemini generateContent를 쓴다 —
  이는 영상 생성이 아니라 초저가 텍스트다(대본당 ≈5원). 워크플로우 env의 Gemini 키는 **텍스트 전용**.
- **영상 비주얼(정사각형)은 유료 생성 없이** 다음으로 채운다: ① 라인별로 문구에 맞는 **실사 이미지
  자동 소싱**(Pexels·Openverse·Wikimedia) ② **상품 사진/제품 영상**(상품·기능 설명 라인) ③ `assets/memes/`
  **밈 이미지**(punch). 편집 연출(켄번즈·전환·자막 리듬)으로 생동감을 만든다.

## ⭐ 핵심 규칙 — 수정 후 검증 원칙 (항상 적용)

- **코드를 수정하면 반드시 '정상 작동'을 실제로 확인하고 오류를 검증한 뒤에** 완료·커밋한다.
  문법 체크만으로 끝내지 않는다 — 렌더는 **프레임을 뽑아 눈으로 확인**, 로직은 **실행·단위 테스트**,
  파이프라인은 **로컬/CI 실행 로그 확인**. 에러·경고를 함께 점검한다.
- **확인 전에는 "됐다"고 보고하지 않는다.** 환경 한계로 확인 불가하면(예: iOS 재생, 시크릿 필요)
  그 사실을 명시하고, 확인 가능한 부분까지는 반드시 검증한다.
- 커밋은 검증을 통과한 단위로만. 검증 실패 시 고쳐서 다시 검증한 뒤 커밋한다.
- 렌더 산출물은 **CI의 QA 게이트(M6.5, `src/video/qa.py`)**가 기계 검사한다(자막=대본 일치·
  위치 고정·길이·프레임 번인). 게이트 실패 = 릴리스/업로드 차단 — 절대 우회하지 않는다.

## ⭐ 핵심 규칙 — 유머 원칙 (콘텐츠 전반, 2026-07-12 / 2026-07-13 강화)

- **⭐ 코믹 = 생존 (2026-07-13 사용자 확정)**: 이 채널은 발굴 아이템의 **신박함·혁신성을 부각**하되,
  **반드시 코믹해야 살아남는다.** 신기하기만 하고 안 웃기면 실패. 대본·이미지·연출 **모든 요소가 웃음에
  기여**해야 하고, 판단 기준은 "이게 웃긴가? 신박한가?" 둘 다 통과. 톤은 **만담(셋업→펀치 티키타카)**.
- **텍스트(대본)는 드립이 난무하고 배꼽 빠지게 웃기게. 이미지도, 영상도 웃기게.**
  점잖은 설명서 톤이 나오면 실패다 (`config/prompts/script_gen.md`의 만담 리듬·웃음 무기고 참조).
- 단, **제품 소개 구간은 신박하고 획기적이고 날카롭게** — 킬러 스펙을 숫자로 콕 집는다(정보는 정확히).
- 웃음도 가드레일 안에서: 사람 비하 금지, §3.3 금지어·의학 단정 금지, 이모지 금지.
- **⭐ react 추임새 전면 금지 (사용자 확정 핵심규칙, 2026-07-12 — 절대 재도입 금지)**:
  "실화냐", "ㅋㅋㅋ", "???" 같은 **화면 리액션 스티커/추임새 텍스트를 어떤 형태로도 화면에
  띄우지 않는다.** 웃음은 오직 대사(내레이션·자막 문장) 안에서만 만든다. 3중 잠금:
  ① 대본 프롬프트가 출력 금지 ② `sanitize.py`가 남아 있으면 무조건 제거
  ③ `qa.py`가 렌더 플랜에서 1개라도 발견하면 **발행 차단**. 화면 텍스트는 하단 자막 +
  punch 밈 카드 1회, 딱 둘뿐이다.

## ⭐ 핵심 규칙 — 대본이 곧 자막 (2026-07-12 개편)

- 자막 칸은 **대본 생성 시점에 확정**한다: 각 라인의 `subs` 배열(1~3어절, 공백 포함 2~12자,
  의미 단위로 끊음). 계약: `" ".join(subs) == text` (띄어쓰기까지 완전 일치, `sanitize.py`가
  검증·자동복구). **렌더러는 자막을 재분할하지 않는다** — 글자수 재조립이 띄어쓰기 실종·문맥 파괴·
  들쭉날쭉 길이의 근본 원인이었다.
- 화면 배치 체계(framed): 일반 자막은 **하단 검정바 1곳 고정**(y=1580) / 중앙 밈 카드는
  **punch 라인 딱 1회**(framed에선 정사각형 중앙, 물음표 자동 트리거 폐지). 그 외 화면 텍스트는
  없다(react 추임새 금지 — 위 핵심규칙). 상단 검정바의 **채널명·핸들·로고**는 텍스트 규칙 예외인
  '브랜드 아이덴티티'(추임새 아님)로 항상 노출.
- **펀치 밈 카드 배경 — AI 밈 이미지 라이브러리 (2026-07-12)**: punch 밈 카드는 `assets/memes/`의
  **AI 자체 생성 밈 이미지(글자 없는 원본)**를 상황 매칭으로 골라 **배경**에 깐다(`src/video/memes.py` +
  `memes.json`). **글자는 굽지 않고 렌더가 하단 클린존에 얹어** 한글 왜곡을 없앤다(Veo 화면글자 깨짐과
  같은 함정 회피). 이미지는 재사용(최소 생성), 라이브러리 비면 텍스트 카드로 폴백. **카드는 여전히 1회**
  — '화면 텍스트 둘뿐' 규칙 불변(밈 이미지는 텍스트가 아니라 배경).

2. 각 Phase의 완료 기준(DoD) 충족 후에만 다음 Phase 진행.
3. §3 준수 가드레일은 모든 코드·산출물에 무조건 적용 (협상 불가).
4. 스펙이 모호하면 임의 구현하지 말고 사용자에게 질문 후 진행.
5. 모든 코드는 GitHub Actions(ubuntu-latest) 실행 전제 — 로컬 실행 가정 금지 (사용자는 iPad 환경).

## 폴더 격리 (저장소 공통 규칙)

- 이 프로젝트의 모든 파일은 `projects/coupang-shorts-factory/` 하위에만 둔다.
- 유일한 예외: `.github/workflows/shorts-produce.yml`·`shorts-research.yml`·`deploy-shorts-admin.yml`·
  `deploy-store-page.yml` (GitHub가 워크플로우 위치를 루트로 강제).
  - 트리거는 `workflow_dispatch` + `requests/*.json` 전용 push(paths 필터 적용됨).
    push 트리거를 확장할 때도 반드시 `paths: ['projects/coupang-shorts-factory/**']`
    필터를 유지한다.
- 저장소의 다른 프로젝트 폴더·파일·워크플로우는 읽기만 하고 절대 수정하지 않는다.
- 한 커밋은 한 프로젝트 폴더만 수정한다(`.github/`만 공용 허용) — 루트의
  `project-isolation-guard.yml`이 모든 푸시에서 자동 검사한다.

## Git 운영 규칙 — 핵심 5줄

1. main에서 직접 작업한다 (브랜치·PR 금지, 브랜치가 강제되면 세션 내 main 머지까지 완료).
2. 의미 단위마다 커밋하고 커밋 즉시 push 한다.
3. push 직후 `git log origin/main --oneline -1` 로 원격 반영을 확인한다.
4. force push·이력 재작성·커밋 되돌리기 금지.
5. push 실패 시 우회하지 말고 오류 원문을 보고하고 멈춘다.

상세 근거·절차는 `coupang-shorts-pipeline-spec.md` 참조.

## 시크릿 (GitHub Actions Secrets — 코드에 하드코딩 금지)

| 이름 | 용도 |
|---|---|
| `SHORTS_ELEVENLABS_API_KEY` | ElevenLabs TTS (문자 타임스탬프 동시 수신) |
| `SHORTS_TYPECAST_API_KEY` | Typecast TTS (오디오만 → faster-whisper 폴백) |
| `SHORTS_CLOVA_CLIENT_ID` / `SHORTS_CLOVA_CLIENT_SECRET` | 네이버클라우드 CLOVA Voice Premium (오디오만 → 폴백) |
| `SHORTS_PEXELS_API_KEY` (선택) | 배경 영상 자동 확보(`scripts/fetch_assets.py`) — 없어도 동작(그라데이션 폴백) |
| `SHORTS_YTDLP_COOKIES` (선택·최후수단) | 유튜브 링크 추출이 계속 차단될 때만 — cookies.txt 내용. 계정 제재 위험 고지 필수, 기본 미등록 |

- TTS 프로바이더는 `config/settings.yaml`의 `tts.provider`로 선택 (`auto` = 키 자동 감지).
- 어느 프로바이더든 최종 산출은 `audio.mp3` + `timestamps.json` 으로 동일해야 한다 (공통 계약).

## 산출물 규칙

- 영상·음성 산출물(`data/jobs/`, `*.mp4`, `*.mp3`)은 커밋 금지 — Actions Artifacts로만 전달.
- 예외: `assets/backgrounds/*.mp4` (CC0 배경 원본, `.gitignore` 부정 패턴으로 커밋 허용).
