# 반자동 쇼츠 제작 워크플로우 — 설계 (2026-07-12 확정)

> 사용자와 Claude의 **분업(반자동)** 설계. AI 영상 왜곡·비용 문제를 사람 승인 게이트로 우회한다.
> 상위 SSOT는 `coupang-shorts-pipeline-spec.md`. 이 문서는 그 위의 "운영 흐름" 설계다.

## 1. 목표

매일 **아이템 1개 → 유튜브 쇼츠 1개**를 낮은 비용(개당 약 700원)·낮은 왜곡 위험으로 제작.
사람은 **판단·승인**(아이템 선택·클립 확인·최종 업로드)만, Claude는 **생성·조립·업로드**를 맡는다.

## 2. 분업 흐름 (7단계 + 4.5 승인 게이트)

| 단계 | 담당 | 내용 | 구현 상태 |
|---|---|---|---|
| 1 | Claude(자동, 매일) | 신박 아이템 **5개 추천** — 쿠팡有·Veo적합도·벤치마크 포인트 한 줄 | 🟡 리서치 기반 있음, 확장 필요 |
| 2 | 사용자 | 1개 골라 쿠팡에서 제품 사진·상세페이지 캡처 + 파트너스 링크 복사 | 사람 작업 |
| 3 | 사용자 | 관리자 페이지에서 아이템 선택 → 캡처·링크 업로드 | ✅ 있음(#18 + notes) |
| 4 | Claude | 벤치마크 기반 기획·대본·구도·제목·설명·해시태그 + **Veo 오프닝 클립 1개 생성** | 🟡 대본·조립 있음, **Veo 생성 신규** |
| **4.5** | **사용자** | **Veo 원본 클립만 먼저 확인** → OK / 재생성(클립만) | 🔴 신규 |
| 5 | Claude | 클립 승인되면 TTS·자막·썸네일 얹어 최종 쇼츠 조립 | ✅ 있음(pipeline+render) |
| 6 | 사용자 | 완성 쇼츠 확인 → 업로드 버튼 클릭 | 🟡 영상 탭 있음, 승인 버튼 신규 |
| 7 | Claude | 유튜브 업로드(설명 최상단 링크+고지문 자동) | ✅ 있음(youtube.py) |

## 3. 요금제 — 하이브리드 (확정)

오프닝 **1컷만 Veo Lite(5초)**, 나머지 구간은 **무료 스틸 모션**(실사 사진 켄번즈·패럴랙스, 이미 구현).

| 항목 | 원가 |
|---|---|
| Veo 3.1 Lite 5초 무음 1개 ($0.05/초) | $0.25 |
| 대본·기획 (Claude) | $0.15 |
| 음성 (TTS) | $0.10 |
| 조립·업로드 (GitHub Actions) | $0 |
| **쇼츠 1개 합계** | **≈ $0.50 (약 700원)** |

- 재생성(re-roll)은 깨진 **클립 1개만** 다시 = +$0.25. 4.5 게이트가 이 낭비를 최소화.
- 더 아끼려면 **무료형**(Veo 0개, 순수 스틸 모션 ≈ 350원)으로 시작해 반응 좋은 아이템만 Veo 얹기.
- 왜 Lite: 무음 생성이라 오디오값 제외, 가장 싼 티어. 오프닝 1컷은 짧아 품질 약점이 덜 드러남.

## 4. 새로 만들 것 (3종)

### A. Veo 생성 모듈 (핵심 신규) — `src/video/veo.py`(예정)
- Gemini API(`generativelanguage`) **img2video**로 업로드된 제품 사진을 씨앗 삼아 **5초 무음 세로(9:16)** 클립 1개 생성.
- 모델: Veo 3.1 Lite(무음). 프롬프트는 M3 기획에서 도출한 오프닝 컷 묘사 + **형태 유지·글자 왜곡 방지** 문구를 본문에 내장(핵심규칙: 프롬프트는 복붙 단일 블록).
- 산출: `data/jobs/{job}/veo_hero.mp4` → 렌더가 상품 구간 hero로 사용. 실패·미승인 시 스틸 폴백(무료형).
- 시크릿: `SHORTS_GEMINI_API_KEY`.

### B. 4.5 원본 클립 확인 게이트 (신규)
- 파이프라인을 **2단계로 분리**: (1) `--stage clip` = Veo 클립만 생성 → Release/Artifact로 관리자에 노출 → (2) `--stage assemble` = 승인 후 전체 조립.
- 관리자 페이지: 생성된 원본 클립 인라인 재생 + **[승인]/[재생성]** 버튼. 재생성은 클립만 다시.

### C. 추천 대시보드 (확장)
- `src/research/outlier_finder.py`(벤치마크 아웃라이어)를 확장해 **매일 아이템 5개**를 산출:
  각 항목에 **① 벤치마크 영상 조회수·훅 포인트 ② 쿠팡 검색 링크(구매가능 확인용) ③ Veo 적합도(上·中·下)**.
- Veo 적합도 규칙: 화면·잔글씨·복잡 UI 많은 제품 = 下(왜곡 위험), 단순 형태·색·동작 제품 = 上.
- cron(매일 아침)으로 `data/research/items_{날짜}.json` 산출 → 관리자 페이지 카드로 표시 → 2단계 선택.

## 5. 유튜브 정책·알고리즘 리스크 (비판적 파트너 검토)

1. **합성 콘텐츠 고지**: Veo 클립은 AI 합성. 유튜브는 "현실을 오도할 수 있는" 합성만 고지 요구 →
   제품 회전·연출은 저위험이나, 정책 변화 대비 **합성요소 표시 옵션**을 업로드에 둘 수 있게 설계.
2. **대량·반복 콘텐츠(YPP 리스크)**: 매일 자동 생산은 "inauthentic/mass-produced"로 몰릴 수 있음 →
   스펙대로 **영상마다 고유 상품·대본·이미지** 유지가 수익화(YPP) 방어의 핵심. 하이브리드는 실제
   제품 사진을 써서 유리.
3. **Veo Lite 화질**: 최저 티어라 "AI 티"가 날 수 있음. 오프닝 1컷·5초로 노출 최소화하는 이유.

## 6. 구현 순서 (제안)

- **P1**: Veo 생성 모듈(A) — Gemini img2video 단독 검증(로컬/CI, 프레임 확인). 가장 불확실한 것 먼저.
- **P2**: 파이프라인 2단계 분리(clip↔assemble) + 4.5 관리자 승인 UI(B).
- **P3**: 추천 대시보드(C) — 아이템 5개 산출 + 관리자 카드 + 선택.
- **P4**: 최종 업로드 승인 버튼(6단계) 정리 + 링크·고지 설명란 확인.
- 각 단계는 **수정 후 검증 원칙**(실행·프레임 확인) 통과 후 커밋.

## 7. 남은 사용자 작업

- **유튜브 인증 3종**(`SHORTS_YT_REFRESH_TOKEN`/`CLIENT_ID`/`SECRET`) 발급·등록 — Claude가 P4에서
  **발급 화면 순서를 하나씩** 안내(OAuth 리프레시 토큰 1회 발급 → 깃허브 시크릿 저장 → 무인 업로드).
- `SHORTS_GEMINI_API_KEY` 등록(Veo 생성용).
- 미인증 API는 업로드가 자동 비공개 → 당분간 "비공개 업로드 → 폰에서 공개 전환" 반자동.
